## 1. Third-Party Hosting

Check the company's third-party hosting footprint, particularly S3 buckets. Amazon S3 (Simple Storage Service) buckets may contain endpoints, logs, credentials, user data, or source code. Find them using Google dorking with S3's URL formats (`BUCKET.s3.amazonaws.com` or `s3.amazonaws.com/BUCKET`):

- `site:s3.amazonaws.com COMPANY_NAME`
- `site:amazonaws.com COMPANY_NAME`

If the company uses `custom URLs` for its S3 buckets, try more flexible search terms instead. Companies often still place keywords like `aws` and `s3` in their custom bucket URLs, so try these searches:
- `amazonaws s3 COMPANY_NAME`
- `amazonaws bucket COMPANY_NAME`
- `amazonaws COMPANY_NAME`
- `s3 COMPANY_NAME`

`GrayhatWarfare` (https://buckets.grayhatwarfare.com/) is an online search engine you can use to find publicly exposed S3 buckets. It allows you to search for a bucket by using a keyword. Supply keywords related to your target, such as the application, project, or organization name, to find relevant buckets.

Finally, you can try to brute-force buckets by using keywords. `Lazys3` (https://github.com/nahamsec/lazys3/) is a tool that helps you do this. It relies on a wordlist to guess buckets that are permutations of common bucket names. 

Another good tool is `Bucket Stream` (https://github.com/eth0izzle/bucket-stream/), which parses certificates belonging to an organization and finds S3 buckets based on permutations of the domain names found on the certificates. Bucket Stream also automatically checks whether the bucket is accessible, so it saves you time.

---

After finding target S3 buckets, use AWS CLI to test access:

1. Install: `pip install awscli`
2. Configure following Amazon's documentation
3. List contents: `aws s3 ls s3://BUCKET_NAME/`
4. Download files: `aws s3 cp s3://BUCKET_NAME/FILE_NAME /local/path`

Look for leaked API keys, credentials, or personal data—these should be reported immediately as exposed S3 buckets are often vulnerabilities themselves.

Test write permissions with harmless proof-of-concept:
- Upload test file: `aws s3 cp TEST_FILE s3://BUCKET_NAME/`
- Remove test file: `aws s3 rm s3://BUCKET_NAME/TEST_FILE`

This demonstrates write access without tampering with actual company files.

## 2. GitHub Recon

Find relevant GitHub usernames by searching the organization/product names or checking known employees' accounts. Focus on repositories related to your target and note top contributors to discover more repos.

**Key areas to examine:**
- **Issues/Commits**: Look for unresolved bugs, recent changes, and security patches
- **Code section**: Search for vulnerable snippets, authentication functions, input handling
- **Blame/History**: Track file development over time

**Search for sensitive data:**
- Hardcoded secrets (search terms: "key," "secret," "password")
- Configuration files revealing infrastructure details
- Old endpoints and S3 bucket URLs
- Outdated dependencies (check versions for known vulnerabilities)

**Automation tools:**
- **Gitrob**: Finds potentially sensitive files in public repos
- **TruffleHog**: Specializes in discovering secrets via regex and entropy analysis
- **KeyHacks**: Validates found credentials and shows usage methods

Focus on authentication, password reset, state-changing actions, and user input handling code as prime vulnerability targets.


## 3. Wayback Machine

Web archives are digital repositories that store snapshots of websites across time, providing a historical record of their evolution. Among these archives, the Wayback Machine is the most comprehensive and accessible resource for web reconnaissance.

The Wayback Machine, a project by the Internet Archive, has been archiving the web for over two decades, capturing billions of web pages from across the globe. This massive historical data collection can be an invaluable resource for security researchers and investigators.

| Feature |	Description |	Use Case in Reconnaissance |
|---|---|---|
| Historical Snapshots | View past versions of websites, including pages, content, and design changes. |Identify past website content or functionality that is no longer available.|
| Hidden Directories | Explore directories and files that may have been removed or hidden from the current version of the website. | Discover sensitive information or backups that were inadvertently left accessible in previous versions.|
| Content Changes |	Track changes in website content, including text, images, and links. | Identify patterns in content updates and assess the evolution of a website's security posture.|
