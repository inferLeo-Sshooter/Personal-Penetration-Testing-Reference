## Goals
```
Web Reconnaissance - Core Objectives
├── 🎯 1. Identifying Assets
│   ├── Publicly accessible components
│   │   ├── Web pages
│   │   ├── Subdomains
│   │   ├── IP addresses
│   │   └── Associated services
│   ├── Technologies used
│   │   └── CMS, frameworks, libraries
│   └── Mindset: "Map the kingdom before you invade."

├── 🕵️‍♂️ 2. Discovering Hidden Information
│   ├── Inadvertently exposed content
│   │   ├── Backup files (.bak, .old, .zip)
│   │   ├── Config files (.env, .git, wp-config.php)
│   │   └── Internal documentation or endpoints
│   ├── Unindexed paths or dev assets
│   └── Mindset: "Look where they hope you won’t."

├── 🧱 3. Analyzing the Attack Surface
│   ├── Enumerate and fingerprint all components
│   │   ├── Server software
│   │   ├── Headers
│   │   └── Services and ports
│   ├── Look for weaknesses in:
│   │   ├── Technologies
│   │   ├── Configuration
│   │   └── Unpatched software
│   └── Mindset: "See what they exposed. Break what they forgot."

├── 🧠 4. Gathering Intelligence
│   ├── Identify human elements
│   │   ├── Employees
│   │   ├── Email addresses
│   │   └── Social media behavior
│   ├── Discover naming conventions, patterns
│   ├── Prepare for phishing or SE attacks
│   └── Mindset: "Use their people and patterns against them."
```


## Road
```
Web Info Gathering - Pentester Mindset Map
├── 1. First Principles: Recon Overview
│   ├── Passive Recon
│   │   └── No direct interaction with target (OSINT, search engines)
│   ├── Active Recon
│   │   └── Interacts with target (e.g. directory bruteforcing)
│   └── Mindset: "Learn silently first. Touch later."
│
├── 2. DNS & Subdomains
│   ├── Whois (registrar, emails, IP ranges)
│   ├── DNS Records (A, MX, TXT, NS)
│   ├── Zone Transfer (check for misconfigured servers)
│   ├── Subdomain Discovery
│   │   ├── Wordlists (SecLists)
│   │   └── Certificate Transparency (crt.sh, Amass, Censys)
│   ├── Virtual Hosts (vhost fuzzing)
│   └── Mindset: "Find forgotten doors."
│
├── 3. Fingerprinting
│   ├── Detect technologies (CMS, backend, JS libraries)
│   ├── Tools: WhatWeb, Wappalyzer, Burp, Netcat
│   └── Mindset: "Know your enemy’s weapons."
│
├── 4. Crawling
│   ├── Tools: Burp Spider, Gobuster, Dirsearch
│   ├── Important files: robots.txt, sitemap.xml, .git
│   ├── JS parsing (manual or with LinkFinder)
│   └── Mindset: "Every endpoint is a potential exploit."
│
├── 5. Search Engine Discovery
│   ├── Google Dorks (filetype, intitle, site)
│   ├── Other search engines: DuckDuckGo, Shodan
│   └── Mindset: "Let the internet snitch on them."
│
├── 6. Web Archives
│   ├── Archive.org / Wayback Machine
│   ├── Discover old APIs, forms, URLs
│   └── Mindset: "Their past is still live."
│
└── 7. Automating Recon
    ├── Tools: Subfinder, Amass, httpx, hakrawler, gau, Waybackurls
    ├── Chain tools into automation pipelines
    └── Mindset: "Automate everything except your brain."
```


## Checklist
```
Web Reconnaissance - Pentester Mindset Map 
├── 1. Full Passive Recon (*Goal: Avoid touching the target directly*)
│   ├── Search Engine Discovery
│   │   └── Use with search operators (site:, inurl:, filetype:)
│   ├── Certificate Transparency Logs
│   │   ├── crt.sh
│   │   ├── Censys
│   │   └── Shodan
│   ├── Web Archives
│   │   └── Wayback Machine, archive.today
│   └── OSINT / Info Gathering
│       ├── Maltego
│       ├── Hunter.io
│       ├── Email format guessing
│       └── LinkedIn scraping, GitHub dorks
│
├── 2. Semi-Passive Recon (*Goal: Send queries, but nothing noisy like scans*)
│   ├── DNS & Subdomain Enumeration
│   │   ├── whois
│   │   ├── dig
│   │   ├── nslookup
│   │   └── host
│   └── Tools:
│   │   ├── Subfinder
│   │   ├── Amass (passive mode)
│   │   ├── crt.sh scraping
│   └── Mindset: "Low noise, low footprint."
│
├── 3. Fingerprinting
│   ├── Passive Fingerprinting (Stealthy)
│   │   ├── Wappalyzer (browser)
│   │   ├── BuiltWith
│   │   ├── Netcraft
│   │   ├── Censys
│   │   ├── Security Headers / Favicon Hashing
│   │   ├── Observe the Response
│   │   └── Observe tech via Shodan/Censys
│   ├── Active Fingerprinting (Target Touch)
│   │   ├── WhatWeb
│   │   ├── wafw00f
│   │   ├── Nmap (Banner grabbing, -sV)
│   │   ├── Analyzing HTTP Headers / Cookies
│   │   ├── Robots.txt / sitemap.xml / security.txt
│   │   └── Page Content Analysis
│   └── Mindset: "Understand the tech before touching it."
│
├── 4. Active Recon (*Goal: Max coverage at the cost of stealth*)
│   ├── Crawling / Spidering
│   │   ├── robots.txt, sitemap.xml, .git, .env
│   │   ├── OWASP ZAP
│   │   ├── Burp Suite Spider
│   │   └── Scrapy, Hakrawler
│   ├── DNS Zone Transfer
│   │   └── dig AXFR (test misconfig) (very rare to succeed)
│   ├── Subdomain / VHost Discovery
│   │   ├── dnsenum
│   │   ├── dnsrecon
│   │   ├── ffuf, gobuster
│   │   └── VHost Fuzzing (ffuf -H "Host:")
│   ├── Port/Service Enumeration (if in scope)
│   │   ├── Nmap
│   │   ├── RustScan
│   │   ├── Masscan
│   └── Mindset: "Now we knock on doors."
│
└── 5. Automation & Enrichment
    ├── Recon-ng
    ├── Amass (passive/active)
    ├── Spiderfoot
    ├── theHarvester
    ├── httpx, nuclei
    ├── 		*RECOMMEND*: FinalRecon.
    └── Output Processing: cut, sort, uniq, grep, jq, xargs


## Data Handling & Output Chaining
- Use with tools like:
  - `cut`, `sort`, `uniq`, `grep`, `xargs`, `jq`
  - Pipe domains into `httpx`, `nuclei`, etc.

In case of further recon:
- Port Scanning: Nmap, Masscan, Unicornscan
- Vulnerability Scanning: Nessus, OpenVAS, Nikto
- Network Mapping: Traceroute, Nmap
- OS Fingerprinting: Nmap, Xprobe2
- Service Enumeration: Nmap
```


## Note

- When found a subdomain name, try do another subdomain fuzz from that result. (also, remember to add stuff to /etc/hosts file in case there is no DNS).
- When crawling, recommend using: SpiderRecon (with scrapy env).
