

## Source Code Review — What To Look For

Here's your mental checklist. Think of it as **following the data**:

### 1. Where Does User Input Enter?
```php
$_GET['param']
$_POST['param']
$_REQUEST
$_COOKIE
```
```python
request.args.get()
request.form.get()
request.json
```
```javascript
req.query
req.body
req.params
```

> 💡 **Rule #1:** User input is EVIL until proven safe. Find every place the application accepts input from the user — that's your attack surface.

---

### 2. What Happens To That Input?

Trace the input and ask **is it being sanitized?**

| Input goes into... | Potential vulnerability |
|-------------------|------------------------|
| SQL query | SQL Injection |
| File path | LFI/Path Traversal |
| Shell command | RCE/Command Injection |
| HTML output | XSS |
| `eval()` | Code Injection |
| Redirect URL | Open Redirect |
| Deserialization | Insecure Deserialization |

---

### 3. Dangerous Functions — Grep For These

These are your **red flag functions**. When you see them, investigate immediately:

**PHP:**
```php
system()
exec()
shell_exec()
passthru()
eval()
include()
require()
unserialize()
```

**Python:**
```python
os.system()
subprocess.call()
eval()
exec()
pickle.loads()
```

**JavaScript:**
```javascript
eval()
exec()
innerHTML
document.write()
```

> 💡 **Tip:** When I get source code the first thing I do is literally grep for dangerous functions:
> ```bash
> grep -r "system\|exec\|eval\|include\|shell_exec" . 
> ```
> Then I trace backwards — where does the input come from that feeds this function?

---

### 4. Authentication & Authorization

Always check:
```
- Are there checks BEFORE sensitive functions?
- Can I access admin pages without being admin?
- Are tokens/cookies validated properly?
- Is there a password reset flow? (Like this machine!)
- Are there hardcoded credentials?
```

---

> 💡 **The Golden Rule of Code Review:** Trust nothing that comes from the user. Every bug you find is a place where the developer trusted user input they shouldn't have.
