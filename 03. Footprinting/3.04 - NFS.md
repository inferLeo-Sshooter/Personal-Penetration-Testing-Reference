## NFS

| Component | Description |
|-----------|-------------|
| **NFS Protocol** | Sun Microsystems network file system for Linux/Unix; accesses remote filesystems as local |
| **Version Evolution** | **NFSv2**: UDP-based, broad compatibility<br>**NFSv3**: Variable file size, better error reporting<br>**NFSv4**: Kerberos, firewall-friendly, ACLs, stateful protocol |
| **NFSv4 Advantages** | Single port 2049 (TCP/UDP), works through firewalls, includes security improvements |
| **NFSv4.1 Features** | Parallel NFS (pNFS) for cluster deployments, session trunking/multipathing |
| **Underlying Protocol** | ONC-RPC/SUN-RPC on ports 111 (TCP/UDP) using XDR for data exchange |
| **Authentication** | No built-in auth; relies on RPC protocol options and UNIX UID/GID mappings |
| **Security Limitation** | Client/server UID/GID mappings may differ; suitable only for trusted networks |

## Dangerous Settings

|Option	|Description|
|-|-|
|`rw`	|Read and write permissions.|
|`insecure`	|Ports above 1024 will be used.|
|`nohide`	|If another file system was mounted below an exported directory, this directory is exported by its own exports entry.|
|`no_root_squash`|	All files created by root are kept with the UID/GID 0.|

## Footprinting the Service

When footprinting NFS, the TCP ports `111` and `2049` are essential. We can also get information about the NFS service and the host via RPC

Using Nmap:
```
sudo nmap 10.129.14.128 -p111,2049 -sV -sC
sudo nmap --script nfs* 10.129.14.128 -sV -p111,2049
```

- Show Available NFS Shares: `showmount -e 10.129.14.128`
- Mounting NFS Share:

```
$ mkdir target-NFS
$ sudo mount -t nfs 10.129.14.128:/ ./target-NFS/ -o nolock
$ cd target-NFS
$ tree .

.
└── mnt
    └── nfs
        ├── id_rsa
        ├── id_rsa.pub
        └── nfs.share

2 directories, 3 files
```

- List Contents with Usernames & Group Names: `ls -l mnt/nfs/`
- List Contents with UIDs & GUIDs: `ls -n mnt/nfs/`
- It is important to note that if the `root_squash` option is set, we cannot edit the `backup.sh` file even as `root`.
- Unmounting: `sudo umount ./target-NFS`
