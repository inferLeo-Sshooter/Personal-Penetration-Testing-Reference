## SMTP

## SMTP Protocol Overview

| **Component** | **Description** |
|---|---|
| **Protocol** | Simple Mail Transfer Protocol (SMTP) - for sending emails in IP networks |
| **Architecture** | Client-server based, operates between email clients/servers or between SMTP servers |

## Ports and Security

| **Port** | **Usage** | **Security** |
|---|---|---|
| **25** | Default SMTP port | Unencrypted (plaintext) |
| **587** | Authenticated users | STARTTLS encryption |
| **465** | Encrypted connections | SSL/TLS encryption |

## SMTP Components

| **Component** | **Full Name** | **Function** |
|---|---|---|
| **MUA** | Mail User Agent | Email client that converts emails into headers and body |
| **MSA** | Mail Submission Agent (Relay server) | Validates email origin and authenticity |
| **MTA** | Mail Transfer Agent | Handles sending/receiving, spam checking, storage, DNS lookups |
| **MDA** | Mail Delivery Agent | Delivers emails to recipient mailboxes |

## Key Features

| **Feature** | **Details** |
|---|---|
| **Authentication** | ESMTP with SMTP-Auth to prevent spam |
| **Encryption** | SSL/TLS encryption to protect data transmission |
| **Vulnerability** | Open Relay Attacks due to misconfigurations |
| **Process Flow** | MUA → MSA → MTA → DNS lookup → Destination MTA → MDA → Mailbox |

## SMTP Disadvantages

| **Disadvantage** | **Description** | **Impact** |
|---|---|---|
| **1. No Delivery Confirmation** | SMTP doesn't return usable delivery confirmation; only English error messages with undelivered headers | Poor delivery tracking and user experience |
| **2. No Authentication** | Users aren't authenticated on connection; sender addresses are unreliable | Enables spam, mail spoofing with fake sender addresses |

## Security Solutions

| **Technology** | **Full Name** | **Purpose** |
|---|---|---|
| **DKIM** | DomainKeys Identified Mail | Email authentication and verification protocol |
| **SPF** | Sender Policy Framework | Prevents email spoofing by validating sender IPs |
| **ESMTP** | Extended SMTP | Enhanced SMTP with security features |

## ESMTP Security Process

| **Step** | **Command/Action** | **Result** |
|---|---|---|
| **1. Greeting** | `EHLO` command | Initiates ESMTP session |
| **2. Encryption** | `STARTTLS` command | Establishes SSL-protected connection |
| **3. Authentication** | `AUTH PLAIN` extension | Safe authentication over encrypted channel |

## Additional Security Measures

| **Measure** | **Action** |
|---|---|
| **Spam Detection** | Suspicious emails rejected or moved to quarantine/spam folder |
| **Connection Encryption** | Entire SMTP connection encrypted via TLS after STARTTLS |
| **Modern Usage** | ESMTP is the standard (when people say "SMTP," they usually mean ESMTP) |


## Footprinting the Service

- The default Nmap scripts include smtp-commands, which uses the EHLO command to list all possible commands that can be executed on the target SMTP server: `sudo nmap 10.129.14.128 -sC -sV -p25`
- We can also use the smtp-open-relay NSE script to identify the target SMTP server as an open relay using 16 different tests: `sudo nmap 10.129.14.128 -p25 --script smtp-open-relay -v`
