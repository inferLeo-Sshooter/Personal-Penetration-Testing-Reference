## IMAP / POP3

## Protocol Comparison

| **Feature** | **IMAP** | **POP3** |
|---|---|---|
| **Full Name** | Internet Message Access Protocol | Post Office Protocol |
| **Email Management** | Online management directly on server | Basic listing, retrieving, deleting only |
| **Folder Support** | Yes - hierarchical mailboxes and structures | No |
| **Multi-client Access** | Yes - synchronization across multiple clients | Limited |
| **Email Storage** | Emails remain on server until deleted | Typically downloads and removes from server |
| **Offline Access** | Limited (some clients offer offline mode) | Yes (emails downloaded locally) |

## IMAP Features & Functionality

| **Aspect** | **Details** |
|---|---|
| **Architecture** | Client-server based with network file system for emails |
| **Connection** | Requires active server connection for management |
| **Multi-user Access** | Multiple users can access email server simultaneously |
| **Email Browsing** | Can browse emails directly on server |
| **Synchronization** | Uniform database across multiple clients |
| **Personal Organization** | Create custom folders and folder structures |

## IMAP Connection Details

| **Component** | **Specification** |
|---|---|
| **Default Port** | 143 (unencrypted) |
| **Encrypted Port** | 993 (SSL/TLS alternative) |
| **Command Format** | Text-based ASCII commands |
| **Authentication** | Username and password required after connection |
| **Command Processing** | Multiple commands can be sent without waiting for confirmation |
| **Command Tracking** | Server responses matched to commands via identifiers |

## Security Considerations

| **Security Aspect** | **Details** |
|---|---|
| **Default Security** | Unencrypted - transmits commands, emails, passwords in plaintext |
| **Encryption Options** | SSL/TLS for encrypted sessions |
| **Port Usage** | Standard port 143 or alternative port 993 for encrypted connections |
| **Server Requirements** | Many servers require encrypted IMAP sessions for security |

## Integration with Other Protocols

| **Protocol** | **Purpose** | **Benefit** |
|---|---|---|
| **SMTP** | Send emails | Sent emails copied to IMAP folder for multi-client access |
| **IMAP Folders** | Store sent messages | All clients access sent emails regardless of sending device |


## IMAP Commands

| **Command** | **Description** |
|---|---|
| `1 LOGIN username password` | User's login authentication |
| `1 LIST "" *` | Lists all directories/mailboxes |
| `1 CREATE "INBOX"` | Creates a mailbox with specified name |
| `1 DELETE "INBOX"` | Deletes a mailbox |
| `1 RENAME "ToRead" "Important"` | Renames a mailbox |
| `1 LSUB "" *` | Returns subset of names from user's `active` or `subscribed` mailboxes |
| `1 SELECT INBOX` | Selects a mailbox to access its messages |
| `1 UNSELECT INBOX` | Exits the selected mailbox |
| `1 FETCH <ID> all` | Retrieves data associated with a specific message in mailbox |
| `1 CLOSE` | Removes all messages with the `Deleted` flag set |
| `1 LOGOUT` | Closes the connection with the IMAP server |

## Command Structure Notes

| **Element** | **Purpose** |
|---|---|
| **Command Prefix** | `1` serves as command identifier for server response tracking |
| **Parameters** | Commands use quoted strings for mailbox names and specific syntax |
| **Message Access** | Must SELECT a mailbox before accessing individual messages |
| **Session Management** | LOGIN required at start, LOGOUT terminates session cleanly |


## POP3 Commands

| **Command** | **Description** |
|---|---|
| `USER username` | Identifies the user |
| `PASS password` | Authentication of the user using password |
| `STAT` | Requests the number of saved emails from the server |
| `LIST` | Requests the number and size of all emails from server |
| `RETR id` | Requests the server to deliver the requested email by ID |
| `DELE id` | Requests the server to delete the requested email by ID |
| `CAPA` | Requests the server to display the server capabilities |
| `RSET` | Requests the server to reset the transmitted information |
| `QUIT` | Closes the connection with the POP3 server |

## POP3 Command Flow

| **Phase** | **Commands** | **Purpose** |
|---|---|---|
| **Authentication** | `USER`, `PASS` | User identification and password verification |
| **Transaction** | `STAT`, `LIST`, `RETR`, `DELE` | Email operations and management |
| **Utility** | `CAPA`, `RSET` | Server capabilities and reset functions |
| **Termination** | `QUIT` | Clean connection closure |

## Command Characteristics

| **Aspect** | **Details** |
|---|---|
| **Command Structure** | Simple text-based commands |
| **ID Reference** | Commands like `RETR` and `DELE` use email ID numbers |
| **Session Based** | Sequential authentication required before email access |
| **Server Response** | Each command receives server confirmation/response |

## Footprinting the Service

By default, ports `110` and `995` are used for POP3, and ports `143` and `993` are used for IMAP. The higher ports (`993` and `995`) use TLS/SSL to encrypt the communication between the client and server. Using Nmap, we can scan the server for these ports. 

- `sudo nmap 10.129.14.128 -sV -p110,143,993,995 -sC`

If we successfully figure out the access credentials for one of the employees, an attacker could log in to the mail server and read or even send the individual messages.

- `curl -k 'imaps://10.129.14.128' --user user:p4ssw0rd`
- `curl -k 'imaps://10.129.14.128' --user cry0l1t3:1234 -v`

- **OpenSSL - TLS Encrypted Interaction POP3**: `openssl s_client -connect 10.129.14.128:pop3s`
- **OpenSSL - TLS Encrypted Interaction IMAP**: `openssl s_client -connect 10.129.14.128:imaps`
