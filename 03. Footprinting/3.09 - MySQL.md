## MySQL

## MySQL Overview

| **Aspect** | **Details** |
|---|---|
| **Type** | Open-source SQL relational database management system |
| **Developer/Support** | Oracle |
| **Definition** | Structured collection of data organized for easy use and retrieval |
| **Performance** | Can quickly process large amounts of data with high performance |
| **Architecture** | Client-server principle |

## MySQL Architecture

| **Component** | **Function** |
|---|---|
| **MySQL Server** | Actual database management system |
| **MySQL Clients** | One or more clients that interact with server |
| **Server Responsibilities** | Data storage and distribution |
| **Client Capabilities** | Retrieve and edit data using structured queries |

## Data Organization

| **Element** | **Description** |
|---|---|

## MySQL Command Processing

| **Process Step** | **Details** |
|---|---|
| **Command Translation** | MySQL translates commands into executable code |
| **Action Execution** | Performs requested database operations |
| **Error Handling** | Web application informs user of processing errors |
| **Vulnerability** | SQL injection attacks can provoke error messages |
| **Information Leakage** | Error descriptions may contain important system information |

## SQL Command Capabilities

| **Operation Type** | **Specific Functions** |
|---|---|
| **Data Manipulation** | Display, modify, add, delete rows in tables |
| **Structure Management** | Change table structure |
| **Relationship Management** | Create or delete relationships and indexes |
| **User Management** | Manage database users |
| **Data Processing** | Handle logins, search functions, etc. |

## MySQL vs MariaDB

| **Aspect** | **MySQL** | **MariaDB** |
|---|---|---|
| **Ownership** | Oracle (acquired MySQL AB) | Independent (fork of MySQL) |
| **Origin** | Original database | Fork created by MySQL's chief developer |
| **Code Base** | Original MySQL source code | Based on MySQL source code |
| **Development** | Oracle-controlled | Open-source community |
| **Relationship** | Original system | Often connected/compatible with MySQL |

## Data Flow Process

| **Step** | **Process** |
|---|---|
| **1. Request** | Web application sends SQL commands |
| **2. Processing** | MySQL processes and executes commands |
| **3. Success Response** | Generated information sent back to client |
| **4. Data Delivery** | Data extracts, records, or processing results returned |

## Footprinting the Service

Usually, the MySQL server runs on TCP port `3306`, and we can scan this port with Nmap to get more detailed information.

```
sudo nmap 10.129.14.128 -sV -sC -p3306 --script mysql*
mysql -u root -h 10.129.14.132
mysql -u root -pP4SSw0rd -h 10.129.14.128

mysql> use sys;
mysql> show tables;
mysql> select host, unique_users from host_summary;
```
