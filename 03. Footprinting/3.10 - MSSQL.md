
## MSSQL

## MSSQL Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Microsoft SQL (MSSQL) |
| **Type** | SQL-based relational database management system |
| **Developer** | Microsoft |
| **Source Code** | Closed source (proprietary) |
| **Primary OS** | Initially written for Windows operating systems |

## MSSQL vs MySQL Comparison

| **Feature** | **MSSQL** | **MySQL** |
|---|---|---|
| **Source Code** | Closed source | Open source |
| **Developer** | Microsoft | Oracle (supported) |
| **Primary Platform** | Windows | Cross-platform |
| **Framework Integration** | Strong .NET native support | Various frameworks |

## Platform Support

| **Operating System** | **Support Level** | **Common Usage** |
|---|---|---|
| **Windows** | Primary platform | Most likely encounter |
| **Linux** | Available versions | Less common |
| **MacOS** | Available versions | Less common |
| **Target Environment** | Windows systems | Primary deployment |

## MSSQL Usage Context

| **User Group** | **Reason for Adoption** |
|---|---|
| **Database Administrators** | Strong Windows integration |
| **Developers** | Building .NET framework applications |
| **Microsoft Ecosystem** | Native support for Microsoft technologies |

## MSSQL Clients - SSMS

| **Client Tool** | **Details** |
|---|---|
| **Full Name** | SQL Server Management Studio (SSMS) |
| **Installation Options** | With MSSQL package OR separate download/install |
| **Primary Use** | Initial configuration and long-term database management |
| **User Base** | Database administrators |

## SSMS Installation & Security Implications

| **Installation Aspect** | **Details** | **Security Consideration** |
|---|---|---|
| **Server Installation** | Commonly installed on database server | Direct server access |
| **Client-Side Application** | Can be installed on any management system | **Security Risk** |
| **Admin/Developer Systems** | Used from various management workstations | Multiple potential vulnerabilities |
| **Credential Storage** | May contain saved credentials | **High Risk** |

## Security Vulnerabilities

| **Vulnerability** | **Risk** | **Potential Impact** |
|---|---|---|
| **SSMS on Vulnerable Systems** | Saved credentials exposure | Database access compromise |
| **Client-Side Installations** | Multiple attack vectors | Lateral movement opportunities |
| **Credential Reuse** | Stored authentication data | Unauthorized database access |

## Deployment Characteristics

| **Characteristic** | **Implication** |
|---|---|
| **Windows-Centric** | More likely found in Windows environments |
| **Enterprise Environments** | Common in Microsoft-based infrastructures |
| **Management Distribution** | SSMS clients spread across admin workstations |


Many other clients can be used to access a database running on MSSQL. Including but not limited to:
- `mssql-cli`
- `SQL Server PowerShell`
- `HeidiSQL`
- `SQLPro`
- `Impacket's mssqlclient.py`

Of the MSSQL clients listed above, pentesters may find `Impacket's mssqlclient.py` to be the most useful due to SecureAuthCorp's Impacket project being present on many pentesting distributions at install. 

## Footprinting the Service

```
sudo nmap --script ms-sql-info,ms-sql-empty-password,ms-sql-xp-cmdshell,ms-sql-config,ms-sql-ntlm-info,ms-sql-tables,ms-sql-hasdbaccess,ms-sql-dac,ms-sql-dump-hashes --script-args mssql.instance-port=1433,mssql.username=sa,mssql.password=,mssql.instance-name=MSSQLSERVER -sV -p 1433 10.129.201.248
```
We can also use Metasploit to run an auxiliary scanner called mssql_ping that will scan the MSSQL service and provide helpful information in our footprinting process.

Connecting with `Mssqlclient.py`: `python3 mssqlclient.py Administrator@10.129.201.248 -windows-auth`
