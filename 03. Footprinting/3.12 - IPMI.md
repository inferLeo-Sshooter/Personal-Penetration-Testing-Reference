## IPMI

## IPMI Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Intelligent Platform Management Interface |
| **Type** | Set of standardized specifications for hardware-based host management |
| **Purpose** | System management and monitoring |
| **Independence** | Works independently of BIOS, CPU, firmware, and OS |
| **Subsystem Type** | Autonomous subsystem |

## IPMI Capabilities

| **Capability** | **Details** |
|---|---|
| **System Management** | Manage and monitor systems when powered off or unresponsive |
| **Network Access** | Direct network connection to system hardware |
| **OS Independence** | No login shell or operating system access required |
| **Remote Upgrades** | System upgrades without physical access |

## Primary Use Cases

| **Use Case** | **Scenario** |
|---|---|
| **Pre-Boot Management** | Before OS boots - modify BIOS settings |
| **Powered Down Systems** | When host is fully powered down |
| **System Failure Recovery** | Access host after system failure |

## Monitoring Functions

| **Monitoring Type** | **Specific Parameters** |
|---|---|
| **Environmental** | System temperature, voltage, fan status |
| **Power Management** | Power supplies monitoring |
| **System Information** | Inventory information, hardware logs |
| **Alerting** | SNMP-based alert notifications |

## IPMI Protocol History & Support

| **Aspect** | **Details** |
|---|---|
| **First Published** | Intel, 1998 |
| **Vendor Support** | Over 200 system vendors |
| **Major Vendors** | Cisco, Dell, HP, Supermicro, Intel |
| **Current Version** | IPMI version 2.0 |
| **v2.0 Feature** | Serial over LAN administration, serial console output |

## Required Components

| **Component** | **Function** |
|---|---|
| **BMC** | Baseboard Management Controller - micro-controller, essential IPMI component |
| **ICMB** | Intelligent Chassis Management Bus - chassis-to-chassis communication interface |
| **IPMB** | Intelligent Platform Management Bus - extends the BMC functionality |
| **IPMI Memory** | Stores system event log, repository store data |
| **Communications Interfaces** | Local system, serial, LAN, ICMB, and PCI Management Bus interfaces |

## System Requirements

| **Requirement** | **Details** |
|---|---|
| **Power Source** | IPMI module requires power source |
| **Network Connection** | LAN connection required for proper operation |
| **Host System Status** | Can function even when host system is powered off |

## Administrative Benefits

| **Benefit** | **Application** |
|---|---|
| **Out-of-Band Management** | System access without OS dependency |
| **Remote Administration** | No physical access required |
| **Comprehensive Monitoring** | Hardware-level system oversight |
| **Failure Recovery** | Access during system failures |

## Footprinting the Service

## IPMI Communication

| **Aspect** | **Details** |
|---|---|
| **Protocol Port** | UDP port 623 |
| **Communication Method** | Network-based protocol communication |
| **Protocol Type** | IPMI network protocol |

## BMC (Baseboard Management Controller) Overview

| **Aspect** | **Details** |
|---|---|
| **Definition** | Systems that use the IPMI protocol |
| **Implementation** | Embedded ARM systems running Linux |
| **Connection** | Connected directly to host's motherboard |
| **Installation Options** | Built into motherboards OR added as PCI card |

## Common BMC Implementations

| **Vendor** | **BMC Product** | **Usage Context** |
|---|---|---|
| **HP** | iLO (Integrated Lights-Out) | Common in penetration tests |
| **Dell** | DRAC (Dell Remote Access Controller) | Frequently encountered |
| **Supermicro** | IPMI | Often seen during assessments |

## Server BMC Availability

| **Server Category** | **BMC Status** |
|---|---|
| **Most Servers** | Come with BMC pre-installed |
| **Alternative Servers** | Support adding BMC via PCI card |
| **Enterprise Systems** | Standard BMC implementation |

## BMC Access Capabilities

| **Access Type** | **Control Level** |
|---|---|
| **Full Host Access** | Complete motherboard access |
| **System Monitoring** | Real-time system oversight |
| **Power Management** | Reboot, power off capabilities |
| **System Recovery** | Host operating system reinstallation |
| **Physical Equivalent** | Nearly equivalent to physical system access |

## BMC Access Interfaces

| **Interface Type** | **Implementation** | **Access Method** |
|---|---|---|
| **Web Management Console** | Browser-based interface | HTTP/HTTPS access |
| **Command-Line Access** | Remote terminal access | Telnet or SSH |
| **IPMI Protocol** | Direct IPMI communication | UDP port 623 |

## Security Implications

| **Security Aspect** | **Risk Level** | **Impact** |
|---|---|---|
| **BMC Compromise** | **Critical** | Nearly equivalent to physical access |
| **Motherboard Control** | **High** | Full hardware-level control |
| **System Reinstallation** | **Critical** | Complete system compromise possible |
| **Multiple Access Points** | **High** | Web, SSH/Telnet, and IPMI protocol exposure |

## BMC Deployment Characteristics

| **Characteristic** | **Details** |
|---|---|
| **Embedded Nature** | ARM-based Linux systems |
| **Hardware Integration** | Direct motherboard connection |
| **Vendor Standardization** | Multiple vendors implement similar functionality |
| **Enterprise Standard** | Common in server environments |
