## SSH

## SSH Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Secure Shell (SSH) |
| **Function** | Enables encrypted, direct connection between computers |
| **Standard Port** | TCP 22 |
| **Network Security** | Works within possibly insecure networks |
| **Purpose** | Prevents third-party data interception |

## SSH Platform Support

| **Operating System** | **Implementation** | **Details** |
|---|---|---|
| **Unix/Linux** | Native implementation | Originally Unix application |
| **MacOS** | Native implementation | Built-in support |
| **Windows** | Requires additional program | Not native, needs installation |
| **Cross-Platform** | Universal compatibility | Runs on all common operating systems |

## SSH Versions

| **Version** | **Characteristics** | **Security Level** |
|---|---|---|
| **SSH-1** | Original version | Vulnerable to MITM attacks |
| **SSH-2** | Advanced protocol | Improved encryption, speed, stability, security |
| **Comparison** | SSH-2 vs SSH-1 | SSH-2 not vulnerable to MITM attacks |

## SSH Server Implementations

| **Implementation** | **Details** |
|---|---|
| **OpenSSH** | Open-source fork, common on Linux |
| **Commercial SSH** | Original from SSH Communication Security |
| **OpenBSD Origin** | OpenSSH is OpenBSD project |

## SSH Capabilities

| **Capability** | **Use Case** |
|---|---|
| **Remote Management** | Command line or GUI host management |
| **Command Execution** | Send commands to remote systems |
| **File Transfer** | Transfer files between systems |
| **Port Forwarding** | Tunnel network connections |

## OpenSSH Authentication Methods

| **Method** | **Type** |
|---|---|
| **1. Password Authentication** | Traditional username/password |
| **2. Public-Key Authentication** | Cryptographic key pairs |
| **3. Host-Based Authentication** | Host identity verification |
| **4. Keyboard Authentication** | Interactive keyboard input |
| **5. Challenge-Response Authentication** | Challenge-based verification |
| **6. GSSAPI Authentication** | Generic Security Service API |

## Public Key Authentication Process

| **Step** | **Participant** | **Action** | **Purpose** |
|---|---|---|---|
| **1. Server Authentication** | Server → Client | Sends public host key | Client verifies server identity |
| **2. Key Verification** | Client | Verifies public key against trusted source | Prevents MITM attacks |
| **3. Client Authentication** | Client → Server | Proves access authorization | Establishes user legitimacy |
| **4. Cryptographic Challenge** | Server → Client | Creates problem with client's public key | Tests private key possession |
| **5. Solution Response** | Client → Server | Decrypts with private key, sends solution | Proves legitimate access |

## Public Key Authentication Components

| **Component** | **Location** | **Characteristics** |
|---|---|---|
| **Private Key** | User's computer only | Individual creation, passphrase-protected, always secret |
| **Public Key** | Stored on server | Used for cryptographic challenges |
| **Passphrase** | User input | Longer than typical password, unlocks private key |
| **Key Pair** | Both systems | Unique public-private pair, cannot be forged |

## Authentication Benefits

| **Benefit** | **Details** |
|---|---|
| **Single Passphrase Entry** | Enter once per session for multiple server connections |
| **Session Security** | Logout prevents unauthorized access to connections |
| **No Password Repetition** | Eliminates need to enter password for each server |
| **Physical Security** | Local logout prevents unauthorized local machine access |

## Security Considerations

| **Security Aspect** | **Risk/Protection** |
|---|---|
| **Initial Connection** | Risk of third-party interposition during first contact |
| **Host Key Uniqueness** | Cannot be imitated due to unique key pair |
| **Private Key Protection** | Attacker cannot forge signature without private key access |
| **Session Management** | Proper logout prevents unauthorized access |

## SSH Footprinting

One of the tools we can use to fingerprint the SSH server is `ssh-audit`. It checks the client-side and server-side configuration and shows some general information and which encryption algorithms are still used by the client and server.

```
./ssh-audit.py 10.129.14.132

Change Authentication Method:
ssh -v cry0l1t3@10.129.14.132

For potential brute-force attacks, we can specify the authentication method with the SSH client option PreferredAuthentications.:
ssh -v cry0l1t3@10.129.14.132 -o PreferredAuthentications=password
```

## Rsync

- `Rsync` is a fast and efficient tool for locally and remotely copying files. It can be used to copy files locally on a given machine and to/from remote hosts.
- By default, it uses port 873 and can be configured to use SSH for secure file transfers by piggybacking on top of an established SSH server connection.

## Footprinting Rsync

```
Scanning for Rsync:
sudo nmap -sV -p 873 127.0.0.1

Probing for Accessible Shares:
nc -nv 127.0.0.1 873

Enumerating an Open Share:
rsync -av --list-only rsync://127.0.0.1/dev
```

## R-Services

## R-Services Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | R-Services suite |
| **Purpose** | Enable remote access and command execution between Unix hosts |
| **Protocol** | TCP/IP based |
| **Developer** | Computer Systems Research Group (CSRG), UC Berkeley |
| **Historical Status** | De facto standard for Unix remote access |
| **Replacement** | Replaced by SSH due to security flaws |

## R-Services Security Issues

| **Security Flaw** | **Details** | **Risk Level** |
|---|---|---|
| **Unencrypted Transmission** | Information sent in plain text | **Critical** |
| **Network Interception** | Passwords and login info can be intercepted | **High** |
| **MITM Vulnerability** | Susceptible to man-in-the-middle attacks | **High** |
| **Similar to Telnet** | Same fundamental security weaknesses | **Critical** |

## R-Services Port Allocation

| **Port Range** | **Protocol** | **Usage** |
|---|---|---|
| **512** | TCP | R-services communication |
| **513** | TCP | R-services communication |
| **514** | TCP | R-services communication |

## R-Services Access & Usage

| **Access Method** | **Requirements** |
|---|---|
| **R-commands** | Only accessible through r-commands suite |
| **Commercial OS** | Most commonly used by Solaris, HP-UX, AIX |
| **Current Usage** | Less common nowadays, occasionally found in penetration tests |

## R-Commands Suite

| **Command** | **Full Name** | **Basic Function** |
|---|---|---|
| **rcp** | Remote Copy | File/directory copying |
| **rexec** | Remote Execution | Command execution |
| **rlogin** | Remote Login | Remote system login |
| **rsh** | Remote Shell | Remote shell access |
| **rstat** | Remote Statistics | System statistics |
| **ruptime** | Remote Uptime | System uptime info |
| **rwho** | Remote Who | User information |

## Most Commonly Abused R-Commands

| **Command** | **Service Daemon** | **Port** | **Protocol** | **Description** |
|---|---|---|---|---|
| **rcp** | rshd | 514 | TCP | Copy files/directories bidirectionally between systems. Works like `cp` but **no warning for overwriting files** |
| **rsh** | rshd | 514 | TCP | Opens shell on remote machine **without login procedure**. Uses trusted entries in `/etc/hosts.equiv` and `.rhosts` |
| **rexec** | rexecd | 512 | TCP | Run shell commands remotely. Requires username/password via **unencrypted socket**. Overridden by trusted files |
| **rlogin** | rlogind | 513 | TCP | Log into remote Unix-like hosts. Similar to telnet. Authentication overridden by trusted files |

## Authentication & Trust Files

| **File** | **Purpose** | **Security Impact** |
|---|---|---|
| **`/etc/hosts.equiv`** | Contains list of trusted hosts | Grants automatic access without authentication |
| **`.rhosts`** | User-specific trusted hosts | User-level authentication override |
| **Trust Mechanism** | Automatic access for listed hosts | **Major security vulnerability** |

## R-Services Security Risks

| **Risk** | **Details** |
|---|---|
| **No Authentication** | Some commands bypass login procedures |
| **Unencrypted Credentials** | Passwords sent in plain text |
| **File Overwriting** | rcp provides no overwrite warnings |
| **Trust File Abuse** | Trusted host files can be exploited |

## Access Control Issues

| **Issue** | **Details** | **Impact** |
|---|---|---|
| **Primary Concern** | Inherent access control problems | Main reason SSH replaced r-services |
| **Trust-Based Authentication** | Relies on trusted information from remote client | Vulnerable to spoofing |
| **Authentication Bypass** | Uses trust files to bypass normal authentication | Security vulnerability |

## Authentication Methods

| **Method** | **Details** | **Security Level** |
|---|---|---|
| **Default (PAM)** | Pluggable Authentication Modules | Standard authentication |
| **Trust File Bypass** | Via `/etc/hosts.equiv` and `.rhosts` files | **Vulnerable** |

## Trust Configuration Files

| **File** | **Scope** | **Purpose** | **Syntax** |
|---|---|---|---|
| **`/etc/hosts.equiv`** | Global (all users) | System-wide trusted hosts | `<username> <ip/hostname>` |
| **`.rhosts`** | Per-user | User-specific trusted hosts | `<username> <ip/hostname>` |

## Trust File Syntax & Modifiers

| **Syntax Element** | **Format** | **Example** |
|---|---|---|
| **Basic Entry** | `<username> <ip address>` | `htb-student 10.0.17.10` |
| **Hostname Entry** | `<username> <hostname>` | `user workstation` |
| **Wildcard Modifier** | `+` symbol | Specifies "anything" |
| **Wildcard Example** | `+ 10.0.17.10` | Any external user from specified IP |

## Security Vulnerabilities

| **Vulnerability** | **Exploitation** | **Risk Level** |
|---|---|---|
| **Misconfigured Trust Files** | Authenticate as another user without credentials | **Critical** |
| **Code Execution Potential** | Gain code execution through authentication bypass | **Critical** |
| **Wildcard Abuse** | `+` modifier allows broad unauthorized access | **High** |
