## RDP Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Remote Desktop Protocol (RDP) |
| **Developer** | Microsoft |
| **Purpose** | Remote access to Windows operating systems |
| **Functionality** | Display and control commands via GUI over IP networks |
| **OSI Layer** | Application layer in TCP/IP reference model |

## RDP Network Configuration

| **Component** | **Protocol** | **Port** | **Usage** |
|---|---|---|---|
| **Primary Transport** | TCP | 3389 | Standard RDP connections |
| **Alternative Transport** | UDP | 3389 | Remote administration (connectionless) |

## Network Requirements

| **Requirement** | **Details** | **Purpose** |
|---|---|---|
| **Network Firewall** | Must allow external connections | Enable remote access |
| **Server Firewall** | Must permit RDP connections | Allow service access |
| **Public IP (NAT)** | Required when NAT is used | Internet-based connections |
| **Port Forwarding** | Set up on NAT router toward server | Route traffic to correct destination |

## RDP Security Evolution

| **Windows Version** | **Security Feature** | **Details** |
|---|---|---|
| **Windows Vista+** | TLS/SSL support | All data and login process encrypted |
| **Legacy Systems** | RDP Security | Inadequate encryption still accepted |
| **Modern Implementation** | Good encryption | Network-protected data transmission |

## RDP Security Vulnerabilities

| **Vulnerability** | **Details** | **Risk Level** |
|---|---|---|
| **Self-Signed Certificates** | Default certificates are self-signed | **Medium** |
| **Certificate Verification** | Client cannot distinguish genuine from forged certificates | **Medium** |
| **Certificate Warnings** | Users receive certificate warnings | User awareness required |
| **Inadequate Encryption** | Some systems still accept weak RDP Security | **High** |

## RDP Service Characteristics

| **Characteristic** | **Details** |
|---|---|
| **Default Installation** | Installed by default on Windows servers |
| **External Dependencies** | No additional external applications required |
| **Activation Method** | Activated using Server Manager |
| **Default Security Setting** | Requires Network Level Authentication (NLA) |

## Network Level Authentication (NLA)

| **Aspect** | **Details** |
|---|---|
| **Default Setting** | Connections only allowed to hosts with NLA |
| **Security Benefit** | Additional authentication layer |
| **Access Control** | Restricts unauthorized connection attempts |

## RDP Connection Requirements

| **Requirement Type** | **Specific Requirements** |
|---|---|
| **Firewall Configuration** | Both network and server firewalls must allow RDP |
| **NAT Configuration** | Port forwarding setup required |
| **Network Access** | Public IP address needed for Internet connections |
| **Authentication** | Network Level Authentication (default) |

## Security Considerations

| **Security Aspect** | **Status** | **Recommendation** |
|---|---|---|
| **Encryption** | TLS/SSL available since Vista | Use modern encryption |
| **Certificate Management** | Self-signed certificates by default | Implement proper certificates |
| **Legacy Support** | Weak encryption still accepted | Disable RDP Security |
| **User Awareness** | Certificate warnings require user judgment | Train users on certificate validation |

## Footprinting RDP

```
nmap -sV -sC 10.129.201.248 -p3389 --script rdp*
nmap -sV -sC 10.129.201.248 -p3389 --packet-trace --disable-arp-ping -n
```

A Perl script named rdp-sec-check.pl has also been developed by Cisco CX Security Labs that can unauthentically identify the security settings of RDP servers based on the handshakes.

```
RDP Security Check - Installation:
sudo cpan

RDP Security Check:
$ git clone https://github.com/CiscoCXSecurity/rdp-sec-check.git && cd rdp-sec-check
$ ./rdp-sec-check.pl 10.129.201.248

Authentication and connection to such RDP servers can be made in several ways. For example, we can connect to RDP servers on Linux using xfreerdp, rdesktop, or Remmina and interact with the GUI of the server accordingly.

Initiate an RDP Session:
$ xfreerdp /u:cry0l1t3 /p:"P455w0rd!" /v:10.129.201.248
```

## WinRM

## WinRM Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Windows Remote Management (WinRM) |
| **Type** | Simple Windows integrated remote management protocol |
| **Interface** | Command line based |
| **Protocol Foundation** | Uses Simple Object Access Protocol (SOAP) |
| **Connection Method** | Establishes connections to remote hosts and applications |

## WinRM Configuration Requirements

| **Windows Version** | **Configuration Status** | **Details** |
|---|---|---|
| **Windows 10+** | Must be explicitly enabled and configured | Not enabled by default |
| **Windows Server 2012+** | Enabled by default | Requires configuration |
| **Older Server Versions** | Must be configured manually | Requires firewall exceptions |
| **Older Clients** | Must be configured manually | Additional setup needed |

## WinRM Ports & Protocols

| **Port** | **Protocol** | **Encryption** | **Usage** |
|---|---|---|---|
| **5985** | TCP | Unencrypted | Standard WinRM communication |
| **5986** | TCP | HTTPS | Encrypted WinRM communication |

## Historical Port Usage

| **Port** | **Previous Usage** | **Current Status** |
|---|---|---|
| **80** | Previously used for WinRM | Mainly blocked for security reasons |
| **443** | Previously used for WinRM | Replaced by dedicated ports |
| **5985/5986** | Current WinRM ports | Modern implementation |

## WinRM Components

| **Component** | **Full Name** | **Function** |
|---|---|---|
| **WinRM** | Windows Remote Management | Core remote management protocol |
| **WinRS** | Windows Remote Shell | Execute arbitrary commands on remote systems |

## WinRS Characteristics

| **Aspect** | **Details** |
|---|---|
| **Availability** | Included by default on Windows 7+ |
| **Function** | Execute remote commands on other servers |
| **Integration** | Works with WinRM for administration |

## WinRM-Dependent Services

| **Service** | **Function** | **WinRM Dependency** |
|---|---|---|
| **PowerShell Remote Sessions** | Remote PowerShell execution | Requires WinRM |
| **Event Log Merging** | Centralized log management | Requires WinRM |

## WinRM Evolution

| **Aspect** | **Historical** | **Current** |
|---|---|---|
| **Port Usage** | Ports 80/443 | Ports 5985/5986 |
| **Security** | Basic implementation | HTTPS support on 5986 |
| **Default Status** | Manual configuration | Enabled by default (Server 2012+) |

## Configuration Requirements

| **Requirement Type** | **Details** |
|---|---|
| **Firewall Configuration** | Necessary firewall exceptions must be created |
| **Service Enablement** | WinRM service must be started |
| **Network Access** | Proper port configuration required |
| **HTTPS Setup** | Port 5986 for encrypted communications |

## Administrative Capabilities

| **Capability** | **Description** |
|---|---|
| **Remote Command Execution** | Execute commands on remote Windows systems |
| **PowerShell Integration** | Remote PowerShell session management |
| **Event Management** | Centralized event log collection |
| **System Administration** | Comprehensive Windows remote management |

## Footprinting WinRM

 WinRM uses TCP ports `5985 (HTTP)` and `5986 (HTTPS)` by default, which we can scan using Nmap. However, often we will see that only HTTP (`TCP 5985`) is used instead of HTTPS (`TCP 5986`).

 - Nmap: `nmap -sV -sC 10.129.201.248 -p5985,5986 --disable-arp-ping -n`
 - In `Linux-based` environments, we can use the tool called `evil-winrm`, another penetration testing tool designed to interact with WinRM: `evil-winrm -i 10.129.201.248 -u Cry0l1t3 -p P455w0rD!`

## WMI

## WMI Overview

| **Aspect** | **Details** |
|---|---|
| **Full Name** | Windows Management Instrumentation (WMI) |
| **Developer** | Microsoft |
| **Implementation Type** | Microsoft's implementation and extension of CIM |
| **Platform** | Windows platform |

## WMI Relationship to Standards

| **Standard** | **Full Name** | **Relationship to WMI** |
|---|---|---|
| **CIM** | Common Information Model | WMI is Microsoft's implementation and extension |
| **WBEM** | Web-Based Enterprise Management | WMI provides core functionality for Windows |

## WMI Capabilities

| **Access Type** | **Scope** | **Impact** |
|---|---|---|
| **Read Access** | Almost all Windows settings | Comprehensive system information |
| **Write Access** | Almost all Windows settings | Full system configuration control |
| **System Control** | Complete Windows environment | Most critical Windows interface |

## WMI Importance

| **Role** | **Application** | **System Types** |
|---|---|---|
| **Administration** | System management and maintenance | PCs and servers |
| **Remote Maintenance** | Remote Windows computer management | All Windows systems |
| **Critical Interface** | Most important Windows management tool | Windows environment |

## WMI Access Methods

| **Access Method** | **Type** | **Usage** |
|---|---|---|
| **PowerShell** | Command-line scripting | Modern administration |
| **VBScript** | Scripting language | Legacy/automated tasks |
| **WMIC** | Windows Management Instrumentation Console | Command-line tool |

## WMI Architecture

| **Component Type** | **Description** |
|---|---|
| **Multiple Programs** | Not a single program | Distributed functionality |
| **Databases/Repositories** | Various data storage systems | Information and configuration storage |
| **Modular Design** | Component-based architecture | Flexible and extensible |

## WMI Significance

| **Significance Factor** | **Details** |
|---|---|
| **Comprehensive Access** | Nearly complete Windows system control |
| **Administrative Centrality** | Primary tool for Windows management |
| **Remote Capabilities** | Essential for remote Windows administration |
| **Universal Application** | Used across all Windows system types |

## Footprinting WMI

The initialization of the WMI communication always takes place on TCP port `135`, and after the successful establishment of the connection, the communication is moved to a random port. For example, the program `wmiexec.py` from the `Impacket` toolkit can be used for this.

- **WMIexec.py**: `/usr/share/doc/python3-impacket/examples/wmiexec.py Cry0l1t3:"P455w0rD!"@10.129.201.248 "hostname"`
