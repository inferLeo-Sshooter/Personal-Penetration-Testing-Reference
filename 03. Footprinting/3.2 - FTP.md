
## FTP

| Aspect | Details |
|--------|---------|
| **Protocol Type** | Application layer protocol in TCP/IP stack, same as HTTP/POP |
| **Connection Structure** | Two channels: Control (TCP 21) for commands/status codes, Data (TCP 20) for file transfer |
| **Connection Types** | **Active**: Client connects via port 21, server responds to client port (blocked by firewalls)<br>**Passive**: Server announces port for client connection (firewall-friendly) |
| **Authentication** | Requires credentials; supports anonymous FTP for public access with limited options |
| **Security** | Clear-text protocol vulnerable to sniffing |
| **Functionality** | Upload/download files, organize directories, delete files with server status code responses |
| **Error Handling** | Protocol monitors transmission errors; supports resume after connection interruption |

## TFTP

| Aspect | TFTP vs FTP |
|--------|-------------|
| **Complexity** | Simpler than FTP with basic file transfer only |
| **Transport Protocol** | UDP (unreliable) vs FTP's TCP (reliable) |
| **Authentication** | No user authentication vs FTP's credential-based access |
| **Security Features** | None vs FTP's password protection and user management |
| **Access Control** | Based solely on OS file read/write permissions |
| **File Scope** | Only globally readable/writable files in shared directories |
| **Network Usage** | Local and protected networks only due to security limitations |
| **Error Recovery** | UDP-assisted application layer recovery |

- Few commands of **TFTP**:
 
|Commands	|Description|
|-|-|
|connect|	Sets the remote host, and optionally the port, for file transfers.|
|get	|Transfers a file or set of files from the remote host to the local host.|
|put	|Transfers a file or set of files from the local host onto the remote host.|
|quit|	Exits tftp.|
|status|	Shows the current status of tftp, including the current transfer mode (ascii or binary), connection status, time-out value, and so on.|
|verbose	|Turns verbose mode, which displays additional information during file transfer, on or off.|

Unlike the FTP client, TFTP `does not have directory listing` functionality.

## Default Configuration

One of the most used FTP servers on Linux-based distributions is vsFTPd. The default configuration of vsFTPd can be found in /etc/vsftpd.conf, and some settings are already predefined by default.
