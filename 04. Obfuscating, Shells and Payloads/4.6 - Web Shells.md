

## 1. Laudanum

Laudanum is a penetration testing tool repository containing web shell files for various application languages including ASP, ASPX, JSP, and PHP. These files are designed for authorized security testing to establish reverse shells and execute commands during legitimate penetration tests. It comes pre-installed on security-focused distributions like Parrot OS and Kali Linux.

**Working with Laudanum**

Laudanum files are located in `/usr/share/laudanum`. Most files can be copied and deployed directly, but shell files require editing to insert your attacking host's IP address for proper callback functionality. Always review file contents and comments before deployment to ensure correct configuration and usage.

**Laudanum Demonstration**

- First, add Lab's IP and domain to your `/etc/hosts` file.
- Then, Move a Copy for Modification: `$ cp /usr/share/laudanum/aspx/shell.aspx /home/tester/demo.aspx`
- Add your IP address to the `allowedIps` variable on line `59`.
- Take advantage of the target's upload function, upload your file.
- Once the upload is successful, you will need to navigate to your web shell to utilize its functions. 


## 2. Antak Webshell

- Antak is a web shell built in ASP.Net included within the Nishang project. Nishang is an Offensive PowerShell toolset that can provide options for any portion of your pentest.
- Antak utilizes PowerShell to interact with the host, making it great for acquiring a web shell on a Windows server. The UI is even themed like PowerShell.

**Working with Antak**

- The Antak files can be found in the `/usr/share/nishang/Antak-WebShell` directory.

```
[!bash!]$ ls /usr/share/nishang/Antak-WebShell

antak.aspx  Readme.md
```

**Antak Demonstration**

- Add an entry into your `/etc/hosts` file.
- Move a Copy for Modification: `[!bash!][/usr/share/nishang/Antak-WebShell]$ cp /usr/share/nishang/Antak-WebShell/antak.aspx /home/administrator/Upload.aspx`
- Make sure you set credentials for access to the web shell. Modify line 14, adding a user and password.
- This comes into play when you browse to your web shell, much like Laudanum. This can help make your operations more secure by ensuring random people can't just stumble into using the shell.
- Upload and navigate to it to use.

**Web Shell Considerations**

Key issues when using web shells during penetration testing:

* **File Management**: Web applications may auto-delete files after set periods
* **Limited Functionality**: Restricted OS interaction, file navigation, and command chaining (e.g., `whoami && hostname` may fail)
* **Instability**: Non-interactive shells can be unreliable and slow enumeration
* **Detection Risk**: Higher chance of leaving attack evidence

**Operational Security**: For evasive assessments, establish reverse shells quickly and delete payloads to avoid detection. This emulates real attacker behavior and helps test client detection capabilities while preventing post-engagement file discovery.

**Documentation**: Record all methods attempted, success/failure rates, payload names, file locations, and include SHA1/MD5 hashes for verification and attribution in reports.
