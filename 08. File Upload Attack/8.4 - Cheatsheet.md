## Note:
> - File upload vuln is only possible if you found the `upload directory` and your `uploaded file`.
> - In most cases, `upload directory` is `hidden`, and your `uploaded file name` would be `changed`. If fuzzing is not possible, then to exploit, you need to chain it with another vuln to read the actual source code index.php or the page that handle to upload in order to find upload directory.
> - When fuzzing for file `ext`, take notice at ext like: `.svg`, `.xml`, `.html`, ... These files can potentially cause `XXE`, `XSS` which can be used to disclosure info and especially explore `upload directory`. And if this is the case, make sure to validate your request, take a `.svg` file for example:
```
<POST REQUEST HEADERS>
...
...
File-name: file.svg
Content-type: image/svg+xml

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE svg [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<svg>&xxe;</svg>
-----------------------------PREDEFINE Boundary--
```
> - Make sure to validate the follow: `file name extention`: `.svg`, `Content-Type`: `image/svg+xml`, `Request body`: `your payload`.
