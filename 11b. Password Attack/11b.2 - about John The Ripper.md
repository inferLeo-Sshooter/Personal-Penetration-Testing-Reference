

## 1. The basic

John the Ripper (`JtR` or `john`) is a popular open-source password cracking tool for penetration testing, supporting brute-force and dictionary attacks. First released in 1996 for UNIX systems, it's become an industry staple. The `"jumbo"` variant is recommended, offering performance optimizations, multilingual wordlists, and 64-bit support for faster, more accurate cracking. JtR includes conversion tools for various file types and hashes, with regular updates for current security trends.

## 2. Cracking modes

### Single crack mode

`Single crack mode` is a rule-based technique ideal for Linux credentials. It generates password candidates using the victim's username, home directory, and GECOS values (full name, room number, phone number), then applies common string modifications (e.g., user `Bob Smith` might use `Smith1`).

Example `passwd` file:
```
r0lf:$6$ues25dIanlctrWxg$nZHVz2z4kCy1760Ee28M1xtHdGoy0C2cYzZ8l2sVa1kIa8K9gAcdBP.GI6ng/qA4oaMrgElZ1Cb9OeXO4Fvy3/:0:0:Rolf Sebastian:/home/r0lf:/bin/bash
```

This reveals username `r0lf`, name `Rolf Sebastian`, and home directory `/home/r0lf`. Single crack mode uses this data to generate and test candidates:


### Wordlist mode

`Wordlist mode` performs dictionary attacks by testing all passwords from a supplied wordlist against the hash. Basic syntax:

```shell-session
inferLeo@htb[/htb]$ john --wordlist=<wordlist_file> <hash_file>
```

Wordlist files must be plain text with one word per line. Multiple wordlists can be comma-separated. Use `--rules` to apply transformations like appending numbers, capitalizing letters, or adding special characters to generate additional candidates.

### Incremental mode

`Incremental mode` is a powerful brute-force-style attack that generates candidates using statistical models (Markov chains). It tests all character combinations from a specific set, prioritizing likely passwords based on training data. This exhaustive but time-consuming mode dynamically generates guesses without wordlists, making it more efficient than naive brute-force through educated statistical predictions.

Basic syntax:
```shell-session
inferLeo@htb[/htb]$ john --incremental <hash_file>
```

**Note**: This mode can be resource-intensive and slow, especially for long or complex passwords. Customizing the character set and length can improve performance and focus the attack.

By default, JtR uses predefined incremental modes specified in its configuration file (`john.conf`), which define character sets and password lengths. You can customize these or define your own to target passwords that use special characters or specific patterns.

```
inferLeo@htb[/htb]$ grep '# Incremental modes' -A 100 /etc/john/john.conf

# Incremental modes

# This is for one-off uses (make your own custom.chr).
# A charset can now also be named directly from command-line, so no config
# entry needed: --incremental=whatever.chr
[Incremental:Custom]
File = $JOHN/custom.chr
MinLen = 0

# The theoretical CharCount is 211, we've got 196.
[Incremental:UTF8]
File = $JOHN/utf8.chr
MinLen = 0
CharCount = 196

# This is CP1252, a super-set of ISO-8859-1.
# The theoretical CharCount is 219, we've got 203.
[Incremental:Latin1]
File = $JOHN/latin1.chr
MinLen = 0
CharCount = 203

[Incremental:ASCII]
File = $JOHN/ascii.chr
MinLen = 0
MaxLen = 13
CharCount = 95

...SNIP...
```
