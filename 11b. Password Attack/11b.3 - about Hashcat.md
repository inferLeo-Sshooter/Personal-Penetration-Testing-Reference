## 1. The basic

Hashcat is a cross-platform password cracking tool with excellent GPU support. Originally proprietary (2009-2015), it's now open-source and supports multiple attack modes for various hash types.

Basic syntax:
```shell-session
$ hashcat -a 0 -m 0 <hashes> [wordlist, rule, mask, ...]
```

Parameters:
* `-a` - attack mode
* `-m` - hash type  
* `<hashes>` - hash string or file containing hashes
* `[wordlist, rule, mask, ...]` - additional arguments based on attack mode

## 3. Attack modes

- Hashcat has many different attack mode, including `dictionary`, `mask`, `combinator`, and `association`.


### Dictionary attack

- Dictionary attack (`-a 0`) tests each word from a provided wordlist against password hashes until a match is found or the list is exhausted.

### Mask attack

Mask attack (`-a 3`) is a brute-force attack where users define the keyspace explicitly. For example, testing six letters followed by two numbers instead of all possible eight-character combinations.

Masks use symbols representing character sets:

| Symbol | Charset |
|--------|---------|
| ?l | abcdefghijklmnopqrstuvwxyz |
| ?u | ABCDEFGHIJKLMNOPQRSTUVWXYZ |
| ?d | 0123456789 |
| ?h | 0123456789abcdef |
| ?H | 0123456789ABCDEF |
| ?s | «space»!"#$%&'()*+,-./:;<=>?@[]^_`{ |
| ?a | ?l?u?d?s |
| ?b | 0x00 - 0xff |

Custom charsets can be defined with `-1`, `-2`, `-3`, `-4` arguments and referenced as `?1`, `?2`, `?3`, `?4`.

Let's say that we specifically want to try passwords which start with an uppercase letter, continue with four lowercase letters, a digit, and then a symbol. The resulting hashcat mask would be `?u?l?l?l?l?d?s`.
