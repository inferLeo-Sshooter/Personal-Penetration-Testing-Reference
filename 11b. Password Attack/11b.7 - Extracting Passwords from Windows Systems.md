

# 1. Windows Authentication Process

Windows client authentication uses multiple modules for logon, credential retrieval, and verification. `Kerberos` is among the most widely used and complex authentication mechanisms. The Local Security Authority (`LSA`) is a protected subsystem that authenticates users, manages local logins, handles local security, and translates between usernames and security identifiers (SIDs).

The security subsystem maintains security policies and user accounts. On Domain Controllers, these apply domain-wide and are stored in Active Directory. LSA also provides access control, permission checks, and security audit message generation.
