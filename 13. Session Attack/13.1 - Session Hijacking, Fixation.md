

# 1. Session Hijacking

## 1.1 - The basic

Session hijacking occurs when the attacker takes advantage of insecure session identifiers, finds a way to obtain them, and uses them to authenticate to the server and impersonate the victim.

An attacker can obtain a victim's session identifier using several methods, with the most common being:
- Passive Traffic Sniffing
- Cross-Site Scripting (XSS)
- Browser history or log-diving
- Read access to a database containing session information
- Weak session identifiers may also be vulnerable to brute force or prediction attacks
