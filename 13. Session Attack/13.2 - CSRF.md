

# 1. The basic

**What is CSRF?**
Cross-site request forgery (CSRF) is a web vulnerability where attackers trick users into performing unintended actions by bypassing the same origin policy that normally prevents websites from interfering with each other.

**What is the impact of a CSRF attack?**
CSRF attacks cause users to unknowingly perform actions like changing passwords, email addresses, or transferring funds. This can lead to full account compromise, and if the victim has privileged access, attackers may gain control over the entire application.

**How does CSRF work?**

CSRF attacks require three conditions:

* **A relevant action** - Something valuable to exploit, like changing permissions or passwords
* **Cookie-based session handling** - The app uses only session cookies for authentication with no additional validation
* **Predictable request parameters** - All required parameters can be determined or guessed by the attacker

**CSRF Example**

Consider an email change function:
```
POST /email/change HTTP/1.1
Host: vulnerable-website.com
Cookie: session=yvthwsztyeQkAPzeQ5gHgTvlyxHfsAfE
email=wiener@normal-user.com
```

This is CSRF-vulnerable because:
* Changing email enables account takeover via password reset
* Only session cookies authenticate the request
* The email parameter is predictable

**The Attack:**
The attacker creates a malicious page:
```html
<form action="https://vulnerable-website.com/email/change" method="POST">
  <input type="hidden" name="email" value="pwned@evil-user.net" />
</form>
<script>document.forms[0].submit();</script>
```

When a logged-in victim visits this page, their browser automatically submits the form with their session cookie, changing their email to the attacker's address.
