# Bypassing Referer-based CSRF defenses

Some applications use the HTTP `Referer` header instead of CSRF tokens to defend against attacks by verifying requests originate from their own domain. This approach is generally less effective and often bypassable.

**Referer header**
The HTTP Referer header (misspelled in the HTTP specification) is an optional header containing the URL of the page that linked to the requested resource. Browsers automatically add it when users trigger HTTP requests through links or form submissions. However, various methods allow pages to withhold or modify the Referer value, often for privacy reasons.

# Bypass example

## Validation of Referer depends on header being present

Some applications only validate the `Referer` header when present, skipping validation entirely when it's omitted.

Attackers can exploit this by crafting CSRF attacks that cause browsers to drop the `Referer` header. The easiest method uses a META tag in the attack page:

`<meta name="referrer" content="never">`

```
<html>
  <head>
    <meta name="referrer" content="never">
  </head>
  <body>
   ATTACK FORM HERE
    </form>
    <script>
      document.forms[0].submit();
    </script>
  </body>
</html>
```

## Validation of Referer can be circumvented

Some applications use flawed `Referer` validation that can be bypassed. If validation only checks that the domain starts with the expected value, attackers can use it as a subdomain:

`http://vulnerable-website.com.attacker-website.com/csrf-attack`

If validation simply checks that the `Referer` contains the domain name, attackers can place it elsewhere in the URL:

`http://attacker-website.com/csrf-attack?vulnerable-website.com`

**Note:** This second approach may not work in browsers since many now strip query strings from the `Referer` header by default to prevent data leakage. To override this, ensure your exploit response includes the `Referrer-Policy: unsafe-url` header (note the correct spelling of "Referrer"), which forces browsers to send the full URL including query strings.
