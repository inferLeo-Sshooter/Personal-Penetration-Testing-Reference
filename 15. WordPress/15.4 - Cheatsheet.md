## The Basics

WordPress is a CMS (Content Management System). A CMS includes:

- **Content Management Application (CMA)**: Interface for adding/managing content with WYSIWYG editors
- **Content Delivery Application (CDA)**: Backend that assembles input into functional websites

After installation, all WordPress files and directories are located in the webroot at `/var/www/html`.

### Key WordPress Files

- **`index.php`** - WordPress homepage
- **`license.txt`** - Contains version information
- **`wp-activate.php`** - Handles email activation for new sites
- **`wp-admin/` folder** - Contains admin login page and dashboard. Common login paths include:
  - `/wp-admin/login.php`
  - `/wp-admin/wp-login.php`
  - `/login.php`
  - `/wp-login.php`
  
  *Note: This file can be renamed to obscure the login page*

- **`xmlrpc.php`** - Legacy feature for HTTP/XML data transmission, now replaced by WordPress REST API
- **`wp-config.php`** - Contains WordPress database connection details (name, host, username, password), authentication keys, table prefix, and can enable DEBUG mode for troubleshooting

### Key WordPress Directories

- **`wp-content/`** - Stores plugins, themes, and uploaded files in the `uploads/` subdirectory. These directories should be carefully enumerated as they may contain sensitive data or vulnerabilities that could lead to remote code execution or other exploits.

- **`wp-includes/`** - Contains everything except for the administrative components and the themes that belong to the website. This is the directory where core files are stored, such as certificates, fonts, JavaScript files, and widgets.

## Enumeration

### 1. Core Version Detection

**Page Source Review**: Search for the `meta generator` tag.

```bash
curl -s -X GET http://blog.inlanefreight.com | grep '<meta name="generator"'
```

Links to CSS (stylesheets) and JS (JavaScript) can also provide hints about the version number.

In older WordPress versions, another source for uncovering version information is the `readme.html` file in WordPress's root directory.

### 2. Plugins and Themes Enumeration

**Plugins Enumeration**:
```bash
curl -s -X GET http://blog.inlanefreight.com | sed 's/href=/\n/g' | sed 's/src=/\n/g' | grep 'wp-content/plugins/*' | cut -d"'" -f2
```

**Themes Enumeration**:
```bash
curl -s -X GET http://blog.inlanefreight.com | sed 's/href=/\n/g' | sed 's/src=/\n/g' | grep 'themes' | cut -d"'" -f2
```

The response headers may also contain version numbers for specific plugins.

**Active Plugins Enumeration**:
```bash
curl -I -X GET http://blog.inlanefreight.com/wp-content/plugins/mail-masta
```

The same applies to installed themes.

### 4. User Enumeration

**Methods for user enumeration**:

- **Default credential guessing**
- **Find author of posts**, then try:
  ```bash
  curl http://blog.inlanefreight.com/?author=1
  ```
- **Interact with JSON endpoint** (before version 4.7.1):
  ```bash
  curl http://blog.inlanefreight.com/wp-json/wp/v2/users | jq
  ```

### 5. Login Attacks

**Password brute-forcing attack using `xmlrpc.php`**:

```bash
curl -X POST -d "<methodCall><methodName>wp.getUsersBlogs</methodName><params><param><value>admin</value></param><param><value>CORRECT-PASSWORD</value></param></params></methodCall>" http://blog.inlanefreight.com/xmlrpc.php
```

If the credentials are not valid, you will receive a `403 faultCode error`.

### 6. WPScan

**Basic enumeration**:
```bash
wpscan --url http://blog.inlanefreight.com --enumerate --api-token Kffr4fdJzy9qVcTk<SNIP>
```

**Key parameters**:
- `--enumerate` - Enumerates WordPress components like plugins, themes, and users
- `--enumerate ap` - Enumerates all plugins
- `-t` - Changes the number of threads (default is 5)
- `--api-token` - Supply API token for enhanced scanning

> **Note**: Get your free API token at the official WPScan website, which provides 50 requests daily.
