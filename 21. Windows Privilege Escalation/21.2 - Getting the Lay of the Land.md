

# Situational Awareness

## Network Enumeration
Key things to check:
- **Dual-homed hosts** — multiple NICs may open paths to previously unreachable network segments
- **Routing tables** — reveals local & adjacent networks
- **Domain info** — DC IP addresses if host is AD-joined
- **ARP cache** (`arp -a`) — shows recent host communication, hints at RDP/WinRM targets admins connect to

> Network info may not directly cause privesc but can guide lateral movement or reveal new targets.

### Interface(s), IP Address(es), DNS Information
```
C:\htb> ipconfig /all
```

### ARP Table
```
C:\htb> arp -a
```

### Routing Table
```
C:\htb> route print
```

## Enumerating Protections

### AV / EDR
- Can **interfere with enumeration** and **block/detect** privesc attempts
- Public PoCs are commonly signatured — may need custom/modified payloads
- Identify what's running **before** deploying tools

### AppLocker
- Whitelisting solution — can block `cmd.exe`, `powershell.exe`, specific file types
- Enumerate policies with:
```powershell
Get-AppLockerPolicy -Effective -Xml
```
- Helps determine if an **AppLocker bypass** is needed before running tools

### Key Takeaway
- Some EDRs even block common binaries like `net.exe`, `tasklist`
- Suspicious activity (e.g., `cmd.exe` on an accountant's machine) may trigger alerts
- Early enumeration of defenses = saves time on non-evasive, **critical** on evasive engagements

### Check Windows Defender Status
```
PS C:\htb> Get-MpComputerStatus
```

### List AppLocker Rules
```
PS C:\htb> Get-AppLockerPolicy -Effective | select -ExpandProperty RuleCollections
```

### Test AppLocker Policy
```
PS C:\htb> Get-AppLockerPolicy -Local | Test-AppLockerPolicy -path C:\Windows\System32\cmd.exe -User Everyone

FilePath                    PolicyDecision MatchingRule
--------                    -------------- ------------
C:\Windows\System32\cmd.exe         Denied c:\windows\system32\cmd.exe
```
