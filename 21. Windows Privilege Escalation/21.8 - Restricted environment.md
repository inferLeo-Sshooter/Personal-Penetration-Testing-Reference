# Citrix Breakout

Many organizations use virtualization platforms (Citrix, Terminal Services, AWS AppStream, CyberArk PSM, etc.) for remote access, typically with "lock-down" measures to limit the impact of malicious or compromised accounts. Despite these restrictions, attackers may still "break out" of the environment.

**Basic Breakout Methodology:**
1. Gain access to a **Dialog Box**
2. Exploit it to achieve **command execution**
3. **Escalate privileges** for higher access

In weakly hardened environments, a shortcut to `cmd.exe` may exist in the Start Menu. In stricter setups, searches for `cmd.exe` or `powershell.exe` return nothing, and navigating to `C:\Windows\system32` via File Explorer is blocked. Gaining CMD access in such environments is significant, as it grants broad OS control and enables further privilege escalation.

The below sections cover the most common Citrix breakout techniques.

---

## Bypassing Path Restrictions

<img width="1294" height="1028" alt="image" src="https://github.com/user-attachments/assets/18bf478c-a0f5-47df-af5a-73541fde5189" />

When File Explorer access to `C:\` is blocked by group policy, Windows dialog boxes can serve as a bypass. Once obtained, a dialog box allows navigation to directories containing native executables like `cmd.exe` by entering paths directly into the file name field.

Many Citrix-deployed applications include file interaction features — such as Save, Open, Browse, Import, or Print — that can invoke a Windows dialog box. Common tools like Paint, Notepad, and Wordpad can be used for this purpose.

**Example using MS Paint:**
1. Launch **Paint** from the Start Menu
2. Click **File > Open** to open a dialog box
3. In the file name field, enter the UNC path:
`\\127.0.0.1\c$\users\pmorgan`
4. Set the file type to **All Files** and press Enter

This grants access to the target directory, bypassing the original path restrictions.

## Accessing SMB share from restricted environment

File Explorer restrictions may block direct access to SMB shares, but this can be bypassed using a UNC path within a Windows dialog box.

**Steps:**
1. Start an SMB server on the Ubuntu machine using Impacket's `smbserver.py`
2. In Citrix, open **Paint** → **File** → **Open** to invoke the dialog box
3. Enter the UNC path `\\10.13.38.95\share` in the file name field, set file type to **All Files**, and press **Enter**

<img width="1294" height="1032" alt="image" src="https://github.com/user-attachments/assets/83a68d42-9e85-4022-9007-e3daf1bd031f" />

Since direct file copying via File Explorer is blocked, instead **right-click** the target executable and select **Open** to run it directly from the share.

<img width="1294" height="1032" alt="image" src="https://github.com/user-attachments/assets/aa51234f-341f-49e5-ae80-71b3bad2fe41" />

The `pwn.exe` binary is a simple compiled executable that spawns a CMD console:

```c
#include <stdlib.h>
int main() {
  system("C:\\Windows\\System32\\cmd.exe");
}
```

Once CMD access is obtained, files can be copied from the SMB share to the local directory (e.g., pmorgan's Desktop).

## Alternate Explorer

When File Explorer is heavily restricted, alternative file managers like **Q-Dir** or **Explorer++** can bypass group policy folder restrictions, allowing navigation and access to otherwise blocked directories.

**Explorer++** is particularly recommended for this purpose due to its:
- **Speed** and user-friendly interface
- **Portability** — runs without installation, making it easy to execute directly from an SMB share

<img width="1294" height="1028" alt="image" src="https://github.com/user-attachments/assets/380494dd-c2eb-45f9-a93c-0cb211af52d1" />

As demonstrated, while standard File Explorer cannot copy files from `\\10.13.38.95\share` due to restrictions, Explorer++ successfully facilitates the transfer to pmorgan's Desktop.

## Alternate Registry Editors

When the default Registry Editor is blocked by group policy, alternative GUI-based registry editors can be used to bypass these restrictions. Examples include **SimpleRegedit**, **Uberregedit**, and **SmallRegistryEditor**, all of which allow registry management without being affected by group policy blocking.

## Modify Existing Shortcut File

Windows shortcuts can be modified to point to a desired executable, bypassing folder path restrictions:

1. **Right-click** the shortcut → **Properties**
2. In the **Target** field, replace the path with the desired executable
3. Execute the shortcut to spawn CMD

## Script Execution

If script extensions like `.bat`, `.vbs`, or `.ps` are configured to run automatically with their respective interpreters, a simple script can be used to spawn an interactive console or download and launch third-party applications, bypassing restrictions.

**Example using a batch file:**
1. Create a new text file named `evil.bat`
2. Open it with Notepad and enter `cmd`
3. Save and execute the file

## Escalating Privileges

Once access to the command prompt is established, it's possible to search for vulnerabilities in a system more easily. For instance, tools like [Winpeas](https://github.com/peass-ng/PEASS-ng/tree/master/winPEAS) and [PowerUp](https://github.com/PowerShellEmpire/PowerTools/blob/master/PowerUp/PowerUp.ps1) can also be employed to identify potential security issues and vulnerabilities within the operating system.

## Bypassing UAC

Even though we escalated to admin level rights, accessing the `C:\users\Administrator` directory remains unfeasible due to the presence of `User Account Control (UAC)`. `UAC` is a security mechanism implemented in Windows to protect the operating system from unauthorized changes. With UAC, each application that requires the administrator access token must prompt the end user for consent.

```
C:\Windows\system32> cd C:\Users\Administrator

Access is denied.
```

Numerous [UAC bypass scripts](https://github.com/FuzzySecurity/PowerShell-Suite/tree/master/Bypass-UAC) are available, designed to assist in circumventing the active User Account Control (UAC) mechanism. These scripts offer methods to navigate past UAC restrictions and gain elevated privileges.

```
PS C:\Users\Public> Import-Module .\Bypass-UAC.ps1
PS C:\Users\Public> Bypass-UAC -Method UacMethodSysprep
```
