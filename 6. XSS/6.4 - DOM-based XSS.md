

## 1. What is DOM-Based XSS?

**DOM-based XSS** happens when JavaScript on a page takes untrusted data (like from the URL) and sends it to a dangerous location (like `innerHTML` or `eval()`), causing malicious code to run.

### 🔁 How It Works:
1. **Source**: Attacker-controlled data (e.g., `window.location`).
2. **Sink**: A function or property that executes or renders that data (e.g., `innerHTML`, `eval()`).
3. **Example**: A link like  
   ```
   https://example.com/page?input=<script>alert(1)</script>
   ```
   causes the script to run if the page handles `input` insecurely.

### 💥 Impact:
- JavaScript runs in the victim's browser.
- Can lead to **account hijacking** or other malicious actions.
