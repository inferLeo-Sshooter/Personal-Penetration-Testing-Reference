# XSS between HTML tags

When the XSS context is text between HTML tags, you need to introduce some new HTML tags designed to trigger execution of JavaScript.

Some useful ways of executing JavaScript are:

```
<script>alert(document.domain)</script>
<img src=1 onerror=alert(1)>
```


### 1. Reflected XSS into HTML context with most tags and attributes blocked

This lab contains a reflected XSS vulnerability in the search functionality but uses a web application firewall (WAF) to protect against common XSS vectors.

To solve the lab, perform a cross-site scripting attack that bypasses the WAF and calls the print() function.

**Note:**
> - Upon trying a blocked tag <something> in entry point, we get error 404 with message tag is blocked. This indicate that there a WAF.
> - Since we do get response, try fuzz out a tag that is allowed.
> - **Notice** that unlike Burp/ZAP/..., cli tools like ffuf handle the wordlist different. The `space` in wordlist might not get encode.
> - When found a working tag, try it with a event/attribute: `/input?=<tag event=1>`.
> - We get message that some event and attribute also get blocked. --> Try fuzz out a working one. `/input?=<found-tag FUZZ-here=1>`
> - From here, combine what u found to craft a working payload.

**Note:**

> - Then we can add **attributes, events,...** to the tag.
> - When self test to deliver to victim, if `<iframe>` tag not possible, use `<script> location="payload-here" </script>`

### 3. Lab: Reflected XSS with event handlers and href attributes blocked

This lab contains a reflected XSS vulnerability with some whitelisted tags, but all events and anchor `href` attributes are blocked.

To solve the lab, perform a cross-site scripting attack that injects a vector that, when clicked, calls the `alert` function.

Note that you need to label your vector with the word "Click" in order to induce the simulated lab user to click your vector. For example:

`<a href="">Click me</a>`

**Note:**
> - This lab is about understanding tags and how they can interact with each other.
> - Fuzz out white list that include: `<a>, <animate>`.
> - `<animate>` is an <svg> element --> can use `<svg>`
> - `<a>` can be used to create hyperlinks, but `href` attribute is banned --> bypass with <animate>
> - `<animate>` can be used with `attributeName` and `values`, those can **be used to assign an attribute and their value**. Example: `<animate  attributeName=href values=javascript:alert() />`

### 7. Lab: Reflected XSS in canonical link tag

This lab reflects user input in a canonical link tag and escapes angle brackets.

To solve the lab, perform a cross-site scripting attack on the home page that injects an attribute that calls the alert function.

To assist with your exploit, you can assume that the simulated user will press the following key combinations:
```
ALT+SHIFT+X
CTRL+ALT+X
Alt+X
Please note that the intended solution to this lab is only possible in Chrome
```

**Note:**
> - We can create our own `parameter` by interacting with the url(original url + /? + your parameter + = value): `https://something.../?test=random`
> - Then, this got reflected in source code as a **canonical tag**: `<link rel="canonical" href="https://something.../?test=random">`
> - From here, we can try to add more attribute to that tag to trigger xss (`accesskey="X" onclick="alert(1)"`)
> - During the process, notice that if double quote syntax `"` was url encoded by the web, i tried to use single quote `'` instead, and it worked. I replaced `"` with `'` in my input, when it get reflected in the source code, the `'` turn into `"`.
