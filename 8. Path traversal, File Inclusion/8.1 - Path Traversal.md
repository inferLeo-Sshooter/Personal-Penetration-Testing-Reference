

# 1. Definition

A vulnerability allowing attackers to read arbitrary server files, including:
- Application code/data
- Backend credentials
- Sensitive OS files

In severe cases, attackers may gain write access, enabling application modification and complete server control.

# 2. Reading arbitrary files via path traversal

A shopping app displays product images using:
`<img src="/loadImage?filename=218.png">`

The server reads files from `/var/www/images/` by appending the filename parameter.

**Vulnerability:** Without path traversal protection, attackers can access arbitrary system files:
`https://insecure-website.com/loadImage?filename=../../../etc/passwd`

This transforms the path to:
`/var/www/images/../../../etc/passwd` → `/etc/passwd`

The `../` sequence navigates up directory levels. On Windows, both `../` and `..\` work:
`https://insecure-website.com/loadImage?filename=..\..\..\windows\win.ini`
